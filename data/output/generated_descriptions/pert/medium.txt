**Problem Name:** Project Evaluation and Review Technique (PERT)
**Description:** PERT is a classic constraint programming problem that involves scheduling a set of activities with given durations and dependencies to minimize the total project duration or the finish time of a specific activity.

**Decision Variables:** The main decision variables are `Start`, an array of variables representing the start time of each activity, and `SumTimes`, a variable representing the sum of the start times of all activities. The `Start` array is the core decision variable, as it determines the schedule of the project.

**Constraints:** The key constraints are:
- Precedence constraints: Each activity can only start after all its predecessor activities have finished. This is represented by the constraint `Start[Dependencies[i,1]] >= (Start[Dependencies[i,2]] + Times[Dependencies[i,2]])`, which ensures that an activity starts after its predecessor has finished.
- The dependencies between activities are defined in the `Dependencies` array, where each row represents a dependency between two activities.

**Objective:** The problem's objective is to minimize the finish time of the last activity (`Start[n]`), which represents the total project duration. Alternatively, the problem can be formulated to minimize the sum of the start times of all activities (`SumTimes`), but the provided model focuses on minimizing the finish time of the last activity.
