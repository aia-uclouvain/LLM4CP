Job Shop Scheduling Problem
The Job Shop Scheduling Problem is a classic problem in operations research and computer science that involves scheduling a set of tasks on a set of machines to minimize the total time required to complete all tasks.

The main decision variables in this problem are the start times of each task, which represent when each task begins. The start times are represented by the `start_times` array in the Minizinc model.

The key constraints in this problem are:
- Each task must be completed within its deadline.
- No two tasks can be executed at the same time (disjunctive constraint).
- The end time of each task is the start time plus the duration of the task.

The objective of the Job Shop Scheduling Problem is to minimize the last time, which is the maximum end time of all tasks. In other words, the goal is to find a schedule that completes all tasks as early as possible, while satisfying all the constraints. 

In this specific instance of the problem, there are 12 tasks with different durations and deadlines, and the goal is to find a schedule that minimizes the last time, which is the maximum end time of all tasks. The Minizinc model uses the `disjunctive2` predicate to enforce the disjunctive constraint, which ensures that no two tasks can be executed at the same time. The model also uses the `maximum` function to calculate the last time, which is the maximum end time of all tasks.
