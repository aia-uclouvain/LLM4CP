Newspaper Problem
The Newspaper problem is a classic constraint programming problem that involves scheduling the reading of newspapers by four students, Algy, Bertie, Charlie, and Digby, who share a flat. Each student reads all four newspapers, the Financial Times, the Guardian, the Daily Express, and the Sun, in a specific order and for a specified amount of time.

The main decision variables in this problem are the start and end times for each student to read each newspaper, represented by the `reading_start` and `reading_end` arrays. These variables determine the schedule of newspaper reading for each student.

The key constraints in this problem can be described as follows:
- Each student can only read one newspaper at a time.
- Each newspaper can only be read by one student at a time.
- The reading order for each student must be followed.
- The start time for each student is determined by their wake-up time.
- The end time for each reading session is determined by the start time and the reading time.

The objective of this problem is to find the earliest time at which all students can finish reading all the newspapers, represented by the `earliest_end_time` variable. This is achieved by minimizing the maximum end time of all reading sessions. In other words, the goal is to schedule the reading of newspapers in such a way that all students can finish reading as early as possible.
