name: Number Lock Problem
description: The Number Lock Problem is a classic puzzle where a 3-digit code must be cracked based on a set of given hints. Each hint provides a 3-digit number and a description of how many digits are correct and in the correct position, or correct but in the wrong position. The goal is to find the correct 3-digit code that satisfies all the given hints.
variables: $x = [x_1, x_2, x_3]$, where $x_i$ is a decision variable representing the $i^{th}$ digit of the code, and each $x_i$ is an integer between 0 and 9. These variables represent the digits of the code that we are trying to find.
constraints: $\begin{align*}
&check(y[1,..],x,1,1) \\
&check(y[2,..],x,0,1) \\
&check(y[3,..],x,0,2) \\
&check(y[4,..],x,0,0) \\
&check(y[5,..],x,0,1)
\end{align*}$, where $check(a, b, pos, val)$ is a predicate that checks if the number of correct positions and values between $a$ and $b$ matches $pos$ and $val$ respectively. These constraints ensure that the code $x$ satisfies all the given hints.
objective: The objective of the problem is to satisfy all the constraints, i.e., to find a code $x$ that satisfies all the given hints. Since this is a satisfaction problem, there is no objective function to minimize or maximize. The goal is to find a feasible solution that satisfies all the constraints.
