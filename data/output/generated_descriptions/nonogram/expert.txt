name: Nonogram
description: The Nonogram problem, also known as Painting by Numbers, is a picture logic puzzle where cells in a grid must be colored or left blank according to numbers given at the side of the grid to reveal a hidden picture. The numbers measure how many unbroken lines of filled-in squares there are in any given row or column.
variables: $rows, cols, row\_rule\_len, col\_rule\_len, row\_rules, col\_rules, board, row\_start\_pos, col\_start\_pos$ 
These variables represent the dimensions of the grid ($rows, cols$), the maximum length of the row and column rules ($row\_rule\_len, col\_rule\_len$), the rules for each row and column ($row\_rules, col\_rules$), and the decision variables $board$ (a 2D array where $board[i,j] = 1$ if the cell at row $i$ and column $j$ is filled, and $0$ otherwise), and $row\_start\_pos, col\_start\_pos$ (2D arrays that represent the start position of each rule in the corresponding row or column).
constraints: $\forall i \in 1..rows \: check\_rule(row\_rules[i], board[i], row\_start\_pos[i]) \: \wedge \: \forall j \in 1..cols \: check\_rule(col\_rules[j], board[:,j], col\_start\_pos[j])$ 
These constraints ensure that each row and column satisfies the corresponding rules, where $check\_rule$ is a predicate that checks if a given rule is satisfied by a row or column.
objective: satisfy 
The objective of the problem is to find a solution that satisfies all the constraints, i.e., to find a coloring of the grid that matches the given rules.
