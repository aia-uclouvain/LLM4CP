Nonogram Problem
The Nonogram problem, also known as Painting by Numbers, is a classic constraint programming problem. It involves reconstructing a hidden picture from a set of given clues, which are numbers that represent the lengths of consecutive filled squares in each row and column.

The main decision variables in this problem are the values of the grid cells, represented by the `board` array, where each cell can be either filled (1) or empty (0). Additionally, there are `row_start_pos` and `col_start_pos` arrays, which represent the starting positions of the filled sequences in each row and column, respectively.

The key constraints in this problem can be described as follows: 
- Each row and column must satisfy the given clues, meaning that the lengths of consecutive filled squares must match the corresponding clue numbers.
- There must be at least one blank square between consecutive filled sequences in each row and column.
- The total number of filled squares in each row and column must be equal to the sum of the corresponding clue numbers.

The objective of the Nonogram problem is to find a solution that satisfies all the given constraints, effectively reconstructing the hidden picture. There is no explicit objective function to minimize or maximize; instead, the goal is to find a feasible solution that meets all the constraints.
