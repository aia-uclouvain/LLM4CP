name: Disjunctive Scheduling Problem
description: The Disjunctive Scheduling Problem is a classic problem in constraint programming that involves scheduling a set of tasks with given durations, such that no two tasks overlap in time. The problem requires finding a feasible schedule that satisfies the non-overlap constraint.
variables: $n, origin_i, duration_i, used_i, end\_time_i$ for $i = 1, \ldots, n$. These variables represent the number of tasks ($n$), the start time of each task ($origin_i$), the duration of each task ($duration_i$), a binary variable indicating whether each task is used ($used_i$), and the end time of each task ($end\_time_i$).
constraints: $\forall i \in \{1, \ldots, n\}: used_i = 1 \leftrightarrow duration_i > 0$, $end\_time_i = origin_i + duration_i$ if $used_i = 1$, $end\_time_i = 0$ if $used_i = 0$, and $\forall i, j \in \{1, \ldots, n\}$ with $i < j$: $end\_time_i \leq origin_j$ or $origin_i \geq end\_time_j$ if $used_i = 1$ and $used_j = 1$. These constraints ensure that tasks with non-zero duration do not overlap, and that the end time of each task is calculated correctly.
objective: The objective of the problem is to find a feasible schedule that satisfies the non-overlap constraint, which is achieved by solving the problem with the "satisfy" objective, indicating that the goal is to find a solution that satisfies all the constraints, rather than optimizing a specific objective function.
