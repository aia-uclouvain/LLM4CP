**Job Shop Scheduling Problem**
The Job Shop Scheduling Problem is a classic constraint programming problem that involves scheduling a set of tasks on a single machine, where each task has a specific duration and start time. 

The main decision variables in this problem are: 
- `origin`: the start time of each task
- `duration`: the duration of each task
- `used`: a binary variable indicating whether a task is used or not
- `end_time`: the end time of each task

The key constraints in this problem are:
- Each task with a non-zero duration must be scheduled without overlapping with other tasks
- If a task has a zero duration, it is not used and its end time is zero
- The start time and end time of each task must be consistent with its duration

The objective of this problem is to find a feasible schedule that satisfies all the constraints, rather than minimizing or maximizing a specific objective function. In other words, the goal is to assign start times and durations to each task such that no two tasks overlap, and tasks with zero duration are not used.
