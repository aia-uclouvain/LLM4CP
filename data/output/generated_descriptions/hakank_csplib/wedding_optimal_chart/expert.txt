name: Wedding Seating Chart Problem
description: The Wedding Seating Chart Problem is a classic problem in constraint programming that involves finding an optimal seating arrangement for a wedding. The problem consists of a set of guests with different relationships and a set of tables with limited seating capacity. The goal is to assign each guest to a table such that the total satisfaction of the guests is maximized, where satisfaction is measured by the number of acquaintances each guest has at their table.
variables: $tables, z$, where $tables$ is an array of decision variables $tables[j]$ representing the table assignment for each guest $j$, and $z$ is a decision variable representing the total satisfaction of all guests. These variables determine the seating arrangement and the overall satisfaction of the guests.
constraints: $z = \sum_{j,k \in 1..m, j < k} C[j,k] \cdot \text{bool2int}(tables[j] = tables[k])$, $\forall i \in 1..n: \sum_{j,k \in 1..m, j < k} \text{bool2int}(C[j,k] > 0 \land tables[j] = i \land tables[k] = i) \geq b$, $\forall i \in 1..n: \sum_{j \in 1..m} \text{bool2int}(tables[j] = i) \leq a$, $tables[1] = 1$, where the first constraint calculates the total satisfaction, the second constraint ensures that each table has at least $b$ acquaintances, the third constraint ensures that each table does not exceed its seating capacity $a$, and the fourth constraint is a symmetry-breaking constraint.
objective: Maximize $z$, the total satisfaction of all guests.
