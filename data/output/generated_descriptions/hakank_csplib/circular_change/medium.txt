The Circular Change Problem
The problem is about finding a sequence of values where the number of changes between consecutive values in the sequence meets a certain condition, considering the sequence as a circular list.
The main decision variables are `x`, an array of variables representing the sequence of values, and `c`, a variable representing the number of changes between consecutive values in the sequence. The `x` array represents the sequence of values, and `c` represents the count of changes.
The key constraints are that the number of changes between consecutive values in the sequence, including the change from the last value to the first value (due to the circular nature), must be equal to a given value. The change is determined by a comparison operator, in this case, "not equal to" (`xop = 10` represents `a != b`). This means that every time the value changes from one element to the next (or from the last to the first), it counts as a change.
The objective of the problem is to satisfy the constraint that the number of changes (`c`) is equal to a given value (`4` in the example), rather than minimizing or maximizing a specific value. The model is designed to find a sequence `x` that meets this condition, demonstrating the concept of circular change rather than optimizing for a particular outcome.
