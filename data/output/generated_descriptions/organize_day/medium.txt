Job Shop Scheduling Problem
The Job Shop Scheduling Problem is about organizing a day by scheduling a set of tasks with specific durations, ensuring that certain tasks are completed before others, and that no two tasks overlap in time.

The main decision variables are:
- `Begins`: an array representing the start time of each task
- `Ends`: an array representing the end time of each task
- `Durations`: an array representing the duration of each task

The key constraints are:
- Each task must be completed within a certain time frame (between 9am and 5pm)
- A task cannot start before a certain time (e.g., the first task must start at or after 11am)
- Certain tasks must be completed before others (defined by the `BeforeTasks` array)
- No two tasks can overlap in time (i.e., a task cannot start until the previous task has finished)

The objective of the problem is to find a schedule that satisfies all the constraints, with no specific objective function being minimized or maximized (i.e., the goal is to find a feasible solution). However, alternative objectives are provided, such as minimizing the start time of certain tasks or the total start time of all tasks.
