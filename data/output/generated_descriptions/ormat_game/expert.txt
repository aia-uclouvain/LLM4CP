name: Ormat Game
description: The Ormat Game is a puzzle where a square grid with some colored cells is given, and the goal is to cover all the colored cells using a set of transparent plastic overlays with patterns of black dots, such that each overlay can be used multiple times, and the objective is to use as few overlays as possible.
variables: $x_{1..f}, num\_overlays, problem_{1..n, 1..n}, overlays_{1..f, 1..n, 1..n}$, where $x_{1..f}$ is a binary variable indicating whether overlay $i$ is used or not, $num\_overlays$ is the total number of overlays used, $problem_{1..n, 1..n}$ is a binary matrix representing the grid with colored cells, $overlays_{1..f, 1..n, 1..n}$ is a 3D binary matrix representing the patterns of the overlays.
constraints: $\sum_{o=1}^{f} x_o = num\_overlays$, $\forall i, j \in 1..n: problem_{i, j} = 1 \rightarrow \sum_{o=1}^{f} x_o \* overlays_{o, i, j} \geq 1$, $\forall i, j \in 1..n: problem_{i, j} = 0 \rightarrow \sum_{o=1}^{f} x_o \* overlays_{o, i, j} = 0$, where the first constraint defines the total number of overlays used, the second constraint ensures that each colored cell is covered by at least one overlay, and the third constraint ensures that each blank cell is not covered by any overlay.
objective: minimize $num\_overlays$
