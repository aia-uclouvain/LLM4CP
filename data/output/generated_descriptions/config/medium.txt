**Configuration Problem**
The problem is about configuring a set of racks with different models, each with its own power, connector, and price characteristics, to accommodate a set of cards with specific power requirements and quantities.

**Decision Variables**
The main decision variables are:
- `rack`: an array of variables representing the model assigned to each rack.
- `counters`: a 2D array of variables representing the number of cards of each type assigned to each rack.
- `cost`: a variable representing the total cost of the configuration.

**Key Constraints**
The key constraints are:
- Each rack's power capacity must not be exceeded by the total power required by the cards assigned to it.
- Each rack's connector capacity must not be exceeded by the total number of cards assigned to it.
- The total number of each type of card assigned to all racks must equal the required quantity.
- The total cost is calculated as the sum of the prices of the models assigned to each rack.

**Objective**
The objective is to minimize the total cost of the configuration while satisfying all the constraints. In other words, the goal is to find the cheapest way to configure the racks to accommodate all the cards without exceeding the power and connector capacities of each rack.
