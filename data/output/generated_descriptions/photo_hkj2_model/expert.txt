name: Photo Problem
description: The photo problem is a classic problem in constraint programming where a set of people need to be arranged in a line for a photo, given their preferences to stand next to specific other people. The goal is to satisfy as many of these preferences as possible.
variables: $n, num\_pref, preferences, positions, z$, where $n$ is the number of people, $num\_pref$ is the number of preferences, $preferences$ is a 2D array representing the preferences of each person, $positions$ is an array representing the position of each person in the line, and $z$ is the total number of satisfied preferences. These variables represent the input parameters, the preferences of the people, the arrangement of people in the line, and the objective value.
constraints: $all\_different(positions)$, $z = \sum_{i=1}^{num\_pref} bool2int(nextTo(positions[preferences[i,1]], positions[preferences[i,2]]))$, where the first constraint ensures that each person is assigned a unique position, and the second constraint calculates the total number of satisfied preferences by checking if each preferred pair of people are next to each other in the line.
objective: Maximize $z$, which represents the total number of satisfied preferences, indicating the optimal arrangement of people in the line that satisfies the most preferences.
