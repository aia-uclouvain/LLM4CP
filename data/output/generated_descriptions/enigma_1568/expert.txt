name: Odd Puzzle Problem
description: The Odd Puzzle Problem is a classic constraint programming problem where a multiplication sum has its digits replaced by letters and dots. The goal is to find the six-figure odd number represented by the word "PUZZLE". Different letters stand for different digits, the same letter stands for the same digit, each dot can be any digit, and leading digits cannot be zero. The problem requires finding the values of the letters and dots such that the multiplication sum is valid and the resulting number "PUZZLE" is odd.
variables: $O, D, P, U, Z, L, E, x, num1, num2, num3, num4, ODD, PUZZLE$ - These are the decision variables representing the digits in the multiplication sum. $O, D, P, U, Z, L, E$ represent the letters, $x$ is an array of variables representing the letters, $num1, num2, num3, num4$ represent the numbers in the multiplication sum, $ODD$ represents the number formed by the letters "ODD", and $PUZZLE$ represents the six-figure number formed by the letters "PUZZLE".
constraints: $cp1d(x, [O,D,P,U,Z,L,E])$, $all\_different(x)$, $ODD = 100*O + 10*D + D$, $PUZZLE = 100000*P + 10000*U + 1000*Z + 100*Z + 10*L + E$, $num1 * num2 = num3 + num4*10 + ODD*100$, $PUZZLE = num1 * num2$, $PUZZLE mod 2 = 1$, $num1 * (num2 mod 10) = num3$, $num1 * ((num2 div 10) mod 10) = num4$, $num1 * (num2 div 100) = ODD$ - These constraints ensure that the letters and numbers satisfy the conditions of the problem, including the uniqueness of letters, the formation of the numbers "ODD" and "PUZZLE", the validity of the multiplication sum, and the oddness of the number "PUZZLE".
objective: The objective of the problem is to find a satisfying assignment of values to the decision variables, which is achieved by solving the constraints. There is no explicit objective function to minimize or maximize.
