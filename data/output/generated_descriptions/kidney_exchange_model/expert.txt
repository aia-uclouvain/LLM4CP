name: Kidney Exchange Problem
description: The Kidney Exchange Problem is a classic problem in constraint programming that involves finding the maximum number of possible kidney exchanges between a group of people, where each person has a set of compatible donors. The goal is to maximize the number of exchanges while ensuring that each person receives a kidney from a compatible donor and that the exchange is cyclic, meaning that a person receiving a kidney must also be able to give a kidney.
variables: $x = (x_1, x_2, ..., x_{num\_people})$, where $x_i$ represents the person from whom person $i$ receives a kidney, and $z$ represents the total number of kidney exchanges. These variables represent the assignment of donors to recipients and the total number of exchanges.
constraints: $\forall p \in 1..num\_people (x_p \in compatible\_pruned[p]) \land alldifferent(x)$, which ensures that each person receives a kidney from a compatible donor and that each donor is assigned to at most one recipient.
objective: Maximize $z = \sum_{i=1}^{num\_people} bool2int(x_i \neq i)$, which represents the total number of kidney exchanges. The objective is to maximize the number of exchanges, which means finding the maximum number of possible cycles in the compatibility graph.
