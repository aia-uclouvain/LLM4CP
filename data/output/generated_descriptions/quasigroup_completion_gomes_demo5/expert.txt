name: Quasigroup Completion Problem
description: The Quasigroup Completion Problem is a classic problem in constraint programming that involves completing a partially filled Latin square. A Latin square is an n x n array filled with n different symbols, each occurring exactly once in each row and exactly once in each column. In the Quasigroup Completion Problem, some of the cells in the Latin square are already filled, and the goal is to complete the remaining cells such that the resulting array is a Latin square.
variables: $x_{i,j}$ for $i = 1..n$ and $j = 1..n$, where $x_{i,j}$ represents the value in the $i^{th}$ row and $j^{th}$ column of the Latin square, and is a decision variable that takes a value from the set $\{1, 2, ..., n\}$.
constraints: $\forall i \in \{1..n\}, \forall j \in \{1..n\}, x_{i,j} \in \{1, 2, ..., n\}$, $\forall i \in \{1..n\}, \forall k \in \{1..n\}, \sum_{j=1}^{n} (x_{i,j} = k) = 1$, $\forall j \in \{1..n\}, \forall k \in \{1..n\}, \sum_{i=1}^{n} (x_{i,j} = k) = 1$, where the first constraint ensures that each cell contains a valid symbol, and the second and third constraints ensure that each symbol occurs exactly once in each row and exactly once in each column, respectively.
objective: The Quasigroup Completion Problem is a satisfaction problem, meaning that the objective is to find a feasible solution that satisfies all the constraints, rather than minimizing or maximizing a specific value.
