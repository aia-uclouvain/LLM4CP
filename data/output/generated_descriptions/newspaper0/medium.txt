Newspaper Problem
The Newspaper Problem is a classic constraint programming problem that involves scheduling the reading of newspapers by four students, Algy, Bertie, Charlie, and Digby, who share a flat. Each student reads all four newspapers, the Financial Times, the Guardian, the Daily Express, and the Sun, in a specific order and for a specified amount of time.

The main decision variables in this problem are the start and end times for each student to read each newspaper, represented by the `reading_start` and `reading_end` arrays. These variables determine the schedule of newspaper reading for each student.

The key constraints in this problem can be described in plain language as follows:
- Each student can only start reading a newspaper after they wake up.
- The end time of reading a newspaper is the start time plus the reading time.
- No two students can read the same newspaper at the same time.
- Each student must read the newspapers in a specific order.

The objective of the problem is to find the earliest time at which all students can finish reading all the newspapers, represented by the `earliest_end_time` variable. This is a minimization problem, where the goal is to minimize the `earliest_end_time` by finding the optimal schedule for the students to read the newspapers.
