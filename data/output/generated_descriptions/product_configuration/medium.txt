Product Configuration Problem
The Product Configuration Problem is about selecting the type and quantity of components to include in a product, such that certain requirements are met and the total weight is minimized.

The main decision variables are the array `x`, where `x[i]` represents the number of components of type `i` to include in the product. The `A` matrix represents the characteristics of each component type, including net power generation, disk space, memory capacity, and weight.

The key constraints are:
- Each component type has a minimum and maximum number of components that can be used.
- The total amount of each characteristic (e.g., disk space, memory capacity) must meet or exceed a certain lower bound.
- The number of components of each type must be non-negative.

The objective of the problem is to minimize the total weight of the product, which is calculated by summing the weight of each component multiplied by the number of components of that type. The unit cost of each component is also considered in the calculation.
