name: Cycle Detection Problem
description: The Cycle Detection Problem is a constraint programming problem that involves finding the number of cycles in a given permutation. A cycle is a sequence of elements where each element is mapped to the next element in the sequence, and the last element is mapped back to the first element. The problem also involves finding the number of fixed points in the permutation, which are elements that are mapped to themselves.
variables: $x = (x_1, x_2, ..., x_n)$, where $x_i$ is a decision variable representing the image of element $i$ under the permutation, and $num\_cycles$ is a decision variable representing the total number of cycles in the permutation, and $num\_fixpoints$ is a decision variable representing the total number of fixed points in the permutation. These variables are used to represent the permutation and its properties.
constraints: $all\_different(x)$, $cycle(x, num\_cycles)$, $num\_fixpoints = \sum_{i=1}^{n} (x_i = i)$, where $all\_different(x)$ ensures that each element in the permutation is mapped to a unique image, $cycle(x, num\_cycles)$ ensures that the permutation forms a set of cycles, and $num\_fixpoints = \sum_{i=1}^{n} (x_i = i)$ calculates the number of fixed points in the permutation.
objective: The objective of the problem is to satisfy the given constraints, which means finding a permutation that forms a set of cycles and calculating the number of cycles and fixed points in the permutation. There is no explicit objective function to minimize or maximize.
