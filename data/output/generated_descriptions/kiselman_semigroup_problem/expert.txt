name: Kiselman Semigroup Problem

description: The Kiselman Semigroup Problem is a constraint programming problem that involves finding a sequence of integers drawn from 1 to n, such that between every pair of occurrences of an integer i, there exists an integer greater than i and an integer less than i. The problem is typically interested in counting the number of solutions for a given n.

variables: $x_1, x_2, ..., x_m, len$ 
These are decision variables where $x_i$ represents the i-th element in the sequence, and $len$ represents the length of the sequence. The $x_i$ variables are integers between 0 and n, where 0 is used as a dummy value to represent the end of the sequence.

constraints: $\forall i, j \in \{1, ..., m\}, i < j: (x_i = x_j \land x_i > 0) \rightarrow \exists k, l \in \{i+1, ..., j-1\}: (x_i < x_k \land x_i > x_l)$, 
$\forall i \in \{1, ..., m-1\}: x_i = 0 \rightarrow x_{i+1} = 0$, 
$len = \sum_{i=1}^{m} (x_i > 0)$, 
$len = m$ 
These constraints ensure that between every pair of occurrences of an integer i, there exists an integer greater than i and an integer less than i, that zeros (if any) are always last in the sequence, that the length of the sequence is calculated correctly, and that the sequence is of maximum length.

objective: The objective of the problem is to find a sequence that satisfies all the constraints, which is equivalent to finding a feasible solution. The problem does not have a specific objective function to minimize or maximize, but rather to find all possible solutions that satisfy the constraints.
