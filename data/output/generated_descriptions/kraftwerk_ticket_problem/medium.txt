Kraftwerk Ticket Problem
The Kraftwerk Ticket Problem is a classic constraint programming problem where a group of people want to buy tickets to a series of concerts, with certain constraints on the number of tickets they can buy and the number of concerts they can attend.

The main decision variables in this problem are:
- `buy_concert`: an array that represents which concert each person buys tickets for
- `buy_num_tickets`: an array that represents the number of tickets each person buys
- `attending`: a 2D array that represents which concerts each person attends

The key constraints in this problem are:
- Each person can buy tickets for at most one concert
- Each person can buy at most 6 tickets
- The total number of tickets bought by all people must be equal to the total number of concerts attended by all people
- The number of people attending each concert must be equal to the total number of tickets bought for that concert
- Each person attends the same number of concerts as they buy tickets for

The objective of this problem is to maximize the total happiness of all people, which is calculated as the sum of the preference values of the concerts attended by each person. The preference values are given in a 2D array `prefs`, where `prefs[p, c]` represents the preference value of person `p` for concert `c`. The goal is to find a solution that maximizes the total happiness while satisfying all the constraints.
