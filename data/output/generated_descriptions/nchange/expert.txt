name: NCHANGE Problem
description: The NCHANGE problem is a constraint programming problem that involves finding the number of changes between consecutive variables in an array. The problem is defined by an array of decision variables and a set of constraints that enforce the relationships between these variables. The goal is to determine the number of times the value of the variables changes.
variables: $x = [x_1, x_2, ..., x_n], c\_pos = [c\_pos_1, c\_pos_2, ..., c\_pos\_{n-1}], num\_changes$ - These are the decision variables where $x$ represents the array of variables, $c\_pos$ is a secondary array used to locate the changes, and $num\_changes$ is the total number of changes between consecutive variables in $x$.
constraints: $x = [4,4,3,4,1], 1 \leq x_i \leq n, c\_pos_i = 1 \leftrightarrow x_i \neq x_{i+1}, nchange(num\_changes, x)$ - These constraints define the relationships between the variables, including the initial values of $x$, the bounds of $x$, the relationship between $c\_pos$ and $x$, and the $nchange$ constraint that calculates the number of changes.
objective: satisfy - The objective of this problem is to satisfy all the given constraints, which means finding a solution that adheres to the defined relationships between the variables, rather than minimizing or maximizing a specific value.
