Stochastic General Assignment Problem
The Stochastic General Assignment Problem is a complex scheduling problem that involves assigning tasks to machines and then scheduling these tasks based on their processing times, which are uncertain and can vary across different scenarios. The problem aims to minimize the expected makespan, which is the maximum completion time of all tasks across all machines.

The main decision variables in this problem are:
- `start`: the start time of each task in each scenario
- `dur`: the duration of each task in each scenario
- `b`: a boolean variable indicating whether an optional task is executed or not

The key constraints in this problem can be described as follows:
- Precedence relations: tasks within the same job must be executed in a specific order, with each task starting only after the previous one has been completed.
- Duration constraints: the duration of each task is determined by the optional task executed, and if there are multiple options, only one can be chosen.
- Optional tasks' constraints: for tasks with multiple optional tasks, exactly one option must be chosen, and if there are only two options, they are mutually exclusive.
- Resource constraints: the cumulative duration of tasks executed on each machine cannot exceed its capacity, taking into account the execution of optional tasks.

The objective of the problem is to minimize the expected makespan, which is calculated as the weighted sum of the maximum completion times of all tasks across all scenarios. The weights represent the probability of each scenario occurring. The problem uses a two-stage approach, where the first stage involves assigning tasks to machines, and the second stage involves scheduling these tasks based on their processing times in each scenario.
