Scene Allocation Problem
The scene allocation problem is a classic constraint programming problem that involves deciding when to shoot scenes for a movie. Each scene involves a number of actors, and at most 5 scenes can be filmed per day. The goal is to minimize the production costs by allocating the scenes to the days while ensuring that all actors required for a scene are present on the day the scene is shot.

The main decision variables in this problem are:
- `shoot`: an array of variables representing the day when each scene is shot.
- `isPaid`: a 2D array of variables representing whether an actor is paid on a particular day.
- `isShot`: a 2D array of variables representing whether a scene is shot on a particular day.

The key constraints in this problem can be described as follows:
- Each scene can only be shot once.
- At most 5 scenes can be shot per day.
- If a scene is shot on a particular day, all the actors required for that scene must be paid on that day.
- The day when a scene is shot must be consistent across all the actors and scenes.

The objective of this problem is to minimize the total production cost, which is calculated as the sum of the daily fees of all the actors. The daily fee of an actor is only incurred if the actor is required for a scene that is shot on that day. The problem aims to find an optimal allocation of scenes to days that minimizes the total cost while satisfying all the constraints.
