name: Minesweeper
description: The Minesweeper problem is a classic puzzle where a player is presented with a grid of cells, some of which contain hidden mines. The player's goal is to clear all non-mine cells without detonating a mine. The puzzle provides numerical hints, where each number represents the count of adjacent mine cells. The problem modeled here is a specific instance of the Minesweeper puzzle, where the grid size and the hints are given, and the task is to find the configuration of mines that satisfies the hints.
variables: $x_{i,j} \in \{0,1\}$ for $i \in \{1, \ldots, r\}$ and $j \in \{1, \ldots, c\}$, where $x_{i,j}$ represents whether a mine is present in cell $(i,j)$ or not.
constraints: $\sum_{(i',j') \in N(i,j)} x_{i',j'} = game[i,j]$ for all $(i,j)$ where $game[i,j] \neq X$, where $N(i,j)$ denotes the set of adjacent cells to $(i,j)$. These constraints ensure that the number of mines adjacent to each cell matches the given hints.
objective: The objective of the problem is to find a feasible solution, i.e., an assignment of mines to cells that satisfies all the constraints, as there is no explicit objective function to minimize or maximize.
