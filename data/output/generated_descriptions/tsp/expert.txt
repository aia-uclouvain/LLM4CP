name: Traveling Salesman Problem (TSP)
description: The Traveling Salesman Problem is a classic problem in combinatorial optimization and operations research that involves finding the shortest possible tour that visits a set of cities and returns to the original city. The problem is stated as follows: given a set of cities and their pairwise distances, find the shortest possible tour that visits each city exactly once and returns to the original city.
variables: $x_{ij}, y_{ij}, total$ where $x_{ij}$ is a binary variable that represents whether the salesman travels from city $i$ to city $j$, $y_{ij}$ is an integer variable that represents the number of cars the salesman has after leaving city $i$ and before entering city $j$, and $total$ is an integer variable that represents the total distance of the tour. These variables are used to model the flow of the salesman through the cities and to calculate the total distance of the tour.
constraints: $\sum_{k \in E(i)} x_{k} = 1, \sum_{k \in E(j)} x_{k} = 1, y_{ij} \geq 0, y_{ij} \leq (n-1)x_{ij}, \sum_{k \in E(i)} y_{k} + n\delta_{i1} = \sum_{k \in E(i)} y_{k} + 1$ where $E(i)$ is the set of edges incident to city $i$, $n$ is the number of cities, and $\delta_{i1}$ is a binary variable that is 1 if $i = 1$ and 0 otherwise. These constraints ensure that the salesman visits each city exactly once, that the flow of cars is conserved at each city, and that the salesman starts with $n$ cars at city 1.
objective: minimize $total = \sum_{ij} c_{ij}x_{ij}$ where $c_{ij}$ is the distance between cities $i$ and $j$. The objective is to find the shortest possible tour that visits each city exactly once and returns to the original city.
