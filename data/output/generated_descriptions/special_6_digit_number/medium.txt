The problem is known as the "Special 6 digit number" or "Magic 1/7 series" problem. 
It is about finding a 6-digit number, represented as abcdef, such that when multiplied by 2, 3, 4, 5, and 6, the resulting numbers form a Latin square. 
A Latin square is a table where each row and column contains each digit exactly once.

The main decision variables in this problem are x and y. 
The variable x is a 2D array representing the matrix formed by the numbers, where x[i,j] represents the jth digit of the ith number. 
The variable y is a 1D array representing the numbers themselves, where y[i] is the ith number.

The key constraints in this problem are: 
each row and column in the matrix must contain each digit exactly once (Latin square property), 
each number in the matrix is a multiple of the first number (y[i] = i*y[1]), 
and each number can be represented as a sequence of digits (y[i] = to_num([x[i,j] | j in 1..n])).

The objective of this problem is not to minimize or maximize a specific value, but rather to find a solution that satisfies all the constraints. 
In other words, the goal is to find a 6-digit number that meets the Latin square property when multiplied by 2, 3, 4, 5, and 6. 
The solution to this problem is a specific 6-digit number, which is 142857.
