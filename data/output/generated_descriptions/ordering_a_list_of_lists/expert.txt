name: Lower Triangular Matrix Ordering Problem
description: The problem involves ordering a matrix of rows with distinct values in each row, subject to two constraints: the matrix must be lower triangular with rows ordered in ascending lengths, and values that appear in multiple rows must be in the same column across all rows.
variables: $x_{i,j}, perms_{i,j}$, where $x_{i,j}$ represents the value in the $i^{th}$ row and $j^{th}$ column of the resulting matrix, and $perms_{i,j}$ represents the permutation from the original matrix to the resulting matrix.
constraints: $\forall i \in 1..rows, \forall j \in 1..cols: x_{i,j} \neq 0 \lor x_{i,j} = matrix_{i,j}, \forall i \in 1..rows: alldifferent_except_0([x_{i,j} | j \in 1..cols]), \forall i \in 1..rows: alldifferent([perms_{i,j} | j \in 1..cols]), \forall i \in 1..rows: permutation3([matrix_{i,j} | j \in 1..cols], [perms_{i,j} | j \in 1..cols], [x_{i,j} | j \in 1..cols]), \forall k \in values: \exists j \in 1..cols, \forall i \in value\_rows[k]: x_{i,j} = k$, which ensure that the resulting matrix satisfies the problem constraints, including the lower triangular structure, distinct values in each row, and values in the same column across rows.
objective: Satisfy all constraints, as this is a feasibility problem with no explicit objective function to minimize or maximize.
