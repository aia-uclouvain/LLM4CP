name: Quasigroup Completion Problem (Latin Square Completion)
description: The Quasigroup Completion Problem is a classic problem in constraint programming where the goal is to complete a partially filled table, known as a Latin square or quasigroup, such that each row and column contains each number exactly once. The problem involves arranging a set of numbers into a square array so that no column or row contains the same two numbers, resulting in a complete Latin square.
variables: $x_{i,j}$ for $i,j = 1..n$, where $x_{i,j}$ represents the number in the $i^{th}$ row and $j^{th}$ column of the Latin square, and is a decision variable that can take any value from 1 to $n$.
constraints: $k\_all\_different(x)$, which is equivalent to $\forall i \in \{1..n\} (\forall j \in \{1..n\} (x_{i,j} \neq x_{i,k}) \wedge \forall j \in \{1..n\} (x_{j,i} \neq x_{k,i}))$, ensuring that each row and column contains each number exactly once.
objective: The objective of the problem is to find a feasible solution that satisfies all the constraints, i.e., to complete the Latin square. Since the problem is a satisfaction problem, there is no objective function to minimize or maximize.
