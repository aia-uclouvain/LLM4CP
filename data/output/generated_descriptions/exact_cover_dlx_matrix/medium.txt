Exact Cover Problem
The Exact Cover Problem is a classic constraint programming problem that involves finding a subset of sets that exactly covers a given universe of elements, with each element being covered by exactly one set.
The main decision variables in this problem are the elements of the array `x`, where `x[i]` represents whether the `i-th` set is included in the exact cover or not. The `mat` array represents the relationship between the sets and the elements, where `mat[i, j]` is 1 if the `j-th` element is in the `i-th` set, and 0 otherwise.
The key constraints in this problem are that each element in the universe must be covered by exactly one set. This is represented by the constraint `sum(i in 1..m) (x[i]*mat[i,j]) = 1`, which ensures that for each element `j`, the sum of the products of `x[i]` and `mat[i, j]` is equal to 1. This means that exactly one set that contains the element `j` is included in the exact cover.
The objective of the Exact Cover Problem is to find a subset of sets that satisfies the constraint, which is a feasibility problem. In other words, the goal is to find a solution that exactly covers all elements in the universe, without minimizing or maximizing any specific objective function. The `solve satisfy` statement in the MiniZinc model indicates that the goal is to find a feasible solution that satisfies all the constraints.
