Peg Solitaire
The Peg Solitaire problem is a classic constraint programming problem that involves finding a sequence of moves to transform an initial board configuration into a target configuration. The game is played on a grid where some cells contain pegs, and the goal is to remove pegs by jumping one peg over another.

The main decision variables in this problem are `x` and `y`. `x[i, j, t]` represents the state of the cell at row `i` and column `j` at time `t`, where `0` indicates an empty cell and `1` indicates a cell with a peg. `y[t]` represents the move performed at time `t`, which is a transition from one board configuration to another.

The key constraints in this problem can be described as follows: 
- The initial board configuration is set according to the given initial board.
- The final board configuration is set according to the given final board.
- For each cell, if it is involved in a move at time `t`, its state at time `t+1` is determined by the type of move.
- If a cell is not involved in a move at time `t`, its state remains the same at time `t+1`.

The objective of the Peg Solitaire problem is to find a sequence of moves that transforms the initial board configuration into the target configuration in a specified number of moves, which is determined by the `nMoves` variable. The problem does not explicitly minimize or maximize a particular objective function, but rather seeks to find a feasible solution that satisfies all the constraints.
