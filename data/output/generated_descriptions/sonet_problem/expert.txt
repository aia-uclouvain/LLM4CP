name: SONET problem
description: The SONET problem is a network design problem where the goal is to set up a network between a given number of nodes, with only certain nodes requiring a connection. The nodes are connected by putting them on a ring, and each ring has a limited capacity. The objective is to minimize the number of Add/Drop Multiplexers (ADMs) used, which are required to put a node on a ring.
variables: $r_{ij}$, where $i$ represents the ring and $j$ represents the node, $z$: $r_{ij}$ represents the assignment of nodes to rings, where $r_{ij} = 1$ if node $j$ is assigned to ring $i$ and $r_{ij} = 0$ otherwise. $z$ represents the total number of ADMs used.
constraints: $\forall i, j \in \{1, \ldots, n\}$ where $i < j$, $(d_{ij} = 1) \rightarrow \exists k \in \{1, \ldots, r\} (r_{ki} + r_{kj} \geq 2)$, $\forall k \in \{1, \ldots, r\}$, $\sum_{j=1}^{n} r_{kj} \leq c_k$: The first constraint ensures that if there is a demand between two nodes, they must be assigned to the same ring. The second constraint ensures that the capacity of each ring is not exceeded.
objective: Minimize $z = \sum_{i=1}^{r} \sum_{j=1}^{n} r_{ij}$, which represents the total number of ADMs used.
