name: Graph Partition Problem
description: The Graph Partition Problem is a classic problem in computer science and operations research that involves dividing a graph into a specified number of partitions, such that the number of nodes in each partition is the same, and the number of edges between the partitions is minimized.
variables: $x_1, x_2, ..., x_m$ where $x_i$ is a set of nodes in partition $i$, and $edges\_between$ which is the total number of edges between all partitions. These variables represent the decision variables of the problem, where $x_i$ determines which nodes are assigned to each partition, and $edges\_between$ measures the quality of the partitioning.
constraints: $n \mod m = 0$, $partition\_set(x, 1..n)$, $card(x[i-1]) = card(x[i])$ for $i = 2..m$, $edges\_between = \sum_{i=1}^{n} \sum_{p1,p2 \in 1..m, p1 < p2} (i \in x_{p1}) \* \sum_{j \in nodes[i]} (j \in x_{p2})$, $1 \in x_1$. These constraints ensure that the number of nodes is divisible by the number of partitions, that the nodes are partitioned correctly, that each partition has the same number of nodes, that the number of edges between partitions is calculated correctly, and that node 1 is assigned to the first partition, respectively.
objective: Minimize $edges\_between$, which represents the total number of edges between all partitions. The goal is to find a partitioning of the graph that minimizes the number of edges between the partitions, while ensuring that each partition has the same number of nodes.
