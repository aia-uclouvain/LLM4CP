Job-shop problem
The job-shop problem is a classic problem in operations research and computer science that involves scheduling a set of jobs on a set of machines to minimize the total time required to complete all jobs.

The main decision variables in this problem are the start and end times of each job on each machine, represented by the `job_start` and `job_end` arrays. These variables determine the schedule of jobs on machines.

The key constraints in this problem are:
- Each job must be processed on each machine in a specific order, represented by the `job_order` array.
- Each machine can only process one job at a time, ensuring non-overlaps of jobs on the same machine.
- The start and end times of each job on each machine must satisfy the given job times, represented by the `job_times` array.
- The jobs on each machine must be processed in the order specified by the `job_order` array.

The objective of this problem is to minimize the earliest end time, which represents the minimum time required to complete all jobs on all machines. This is achieved by finding the optimal schedule of jobs on machines that satisfies all the constraints and minimizes the maximum end time of all jobs.
