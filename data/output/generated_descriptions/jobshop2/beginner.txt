Job-shop problem
The job-shop problem is about scheduling jobs on different machines in a way that minimizes the total time it takes to complete all jobs. Imagine a factory with several machines and many tasks to be done on these machines. Each task needs to be done on a specific machine and takes a certain amount of time. The goal is to find the best order for doing these tasks on the machines so that everything is finished as quickly as possible.

The main variables are:
- num_machines: the number of machines available
- num_jobs: the number of tasks to be done
- job_times: how long each task takes on each machine
- job_order: the order in which tasks are done on each machine
- job_start and job_end: when each task starts and ends on each machine

The basic restrictions or rules are:
- Each task must be done on the machines in a specific order (defined by job_order)
- No two tasks can be done on the same machine at the same time
- Each task takes a certain amount of time to complete (defined by job_times)

The goal of the problem is to minimize the earliest end time, which is the time when all tasks on all machines are finished. In other words, the goal is to find a schedule that allows all tasks to be completed as quickly as possible.
