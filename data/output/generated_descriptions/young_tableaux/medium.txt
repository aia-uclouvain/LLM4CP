**Young Tableau Problem**
The Young Tableau problem is a classic constraint programming problem that involves finding a standard Young tableau of a given size. 

**Description**
The problem is about arranging numbers from 1 to n in an n x n grid, such that each row and column is non-decreasing, and each number appears exactly once. The grid represents a Young tableau, which is a mathematical object used to describe the symmetry of a set of objects.

**Decision Variables**
The main decision variables in this problem are:
- `x[i, j]`: a 2D array representing the Young tableau, where `x[i, j]` is the number in the i-th row and j-th column.
- `p[i]`: a 1D array representing the partition structure of the Young tableau, where `p[i]` is the number of elements in the i-th row.

**Constraints**
The key constraints in this problem are:
- Each number from 1 to n appears exactly once in the grid.
- Each row is non-decreasing, meaning that the numbers in each row are in increasing order.
- Each column is non-decreasing, meaning that the numbers in each column are in increasing order.
- The partition structure of the Young tableau is calculated based on the number of elements in each row.
- The sum of the elements in the partition structure is equal to n.
- The partition structure is non-increasing, meaning that the number of elements in each row is greater than or equal to the number of elements in the next row.

**Objective**
The objective of this problem is to find a standard Young tableau that satisfies all the constraints. The problem does not have a specific objective function to minimize or maximize, but rather to find a feasible solution that satisfies all the constraints. The solution is a standard Young tableau, which is a unique arrangement of numbers that satisfies all the constraints.
