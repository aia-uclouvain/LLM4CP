**2D Bin Packing Problem**
The 2D Bin Packing Problem is a classic constraint programming problem that involves packing a set of shapes into a grid, such that each shape is placed within the grid without overlapping with other shapes.

**Decision Variables**
The main decision variables in this problem are:
- `x[i][j]`: the index of the shape occupying the cell at row `i` and column `j` (or -1 if the cell is free)
- `y[k]`: the (index of the) base cell in the grid where we start putting the `k`th shape

**Constraints**
The key constraints in this problem are:
- Each shape must be placed within the grid, without overlapping with other shapes.
- Each cell in the grid can only be occupied by one shape.
- The cells that make up a shape must be contiguous and match the shape's form.
- A shape has exactly the right number of cells mapping to it.

**Objective**
The objective of this problem is to find a feasible packing of all shapes into the grid, such that each shape is placed without overlapping with other shapes. The problem does not have a specific objective function to minimize or maximize, but rather seeks to find a valid solution that satisfies all constraints.
