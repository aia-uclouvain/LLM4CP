2D Bin Packing Problem
The 2D bin packing problem is a classic constraint programming problem that involves packing a set of two-dimensional objects of different shapes and sizes into a bin, such that no two objects overlap and the total area of the bin is minimized.

The main decision variables in this problem are `x` and `kind`, where `x` represents the base position of each object in the bin, and `kind` represents the shape used by each object. The `x` variable is a 2D array, where `x[i,j]` is the position of object `i` in dimension `j`. The `kind` variable is a 1D array, where `kind[i]` is the shape used by object `i`.

The key constraints in this problem are: 
- The `geost_bb` constraint, which ensures that no two objects overlap.
- The constraint that each object must be placed within the bin, i.e., its position must be within the bounds of the bin.
- The constraint that each shape group (A, B, and C) must have one shape variant selected.

The objective of the problem is to minimize the sum of the upper limits of the bin, which represents the total area of the bin. This is achieved by minimizing the `sum_upper` variable, which is the sum of the upper limits of the bin in each dimension.
