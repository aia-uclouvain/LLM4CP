Covering Array Problem
The Covering Array problem is a classic problem in combinatorial design theory and constraint programming. It involves finding a matrix with a specific structure, where each row represents a test case and each column represents a parameter.

The main decision variables in this problem are the elements of the matrix, which represent the values assigned to each parameter in each test case. In the given models, these variables are represented by `CA` in the Essence model and `v` in the PyCSP3 model.

The key constraints in this problem can be described as follows: 
- Each row in the matrix must contain a unique combination of values.
- For any subset of rows of size `t`, all possible combinations of values must be present.
- The matrix must be lexicographically ordered, meaning that each row and column must be in non-decreasing order.

The objective of the Covering Array problem is to find a matrix that satisfies these constraints while minimizing the number of columns (or parameters) required. In other words, the goal is to find the smallest possible matrix that covers all possible combinations of values for any subset of rows of size `t`. 

Note that the problem does not explicitly minimize or maximize a specific objective function, but rather seeks to find a feasible solution that satisfies the given constraints. The size of the matrix (i.e., the number of columns) is implicitly minimized, as a smaller matrix is generally preferred.
