**Problem Name:** Virtual Machine Allocation

**Problem Description:** 
This problem is about allocating different components of a web application to virtual machines (VMs) in a way that minimizes the total cost. The components include things like Wordpress, MySQL, and load balancers. Each component has certain requirements, such as CPU, memory, and storage, and each VM has its own specifications and price. The goal is to assign the components to the VMs in a way that meets all the requirements and minimizes the total cost.

**Main Variables:**

* **AssignmentMatrix:** This is a table that shows which components are assigned to which VMs.
* **VMType:** This is a table that shows the type of each VM.
* **OccupancyVector:** This is a table that shows whether each VM is being used or not.
* **Price:** This is a table that shows the price of each VM.

**Basic Restrictions:**

* Each component must be assigned to a VM that has enough resources (CPU, memory, storage) to meet its requirements.
* Some components, like load balancers, can only be deployed on a single VM.
* Some components, like Varnish, cannot be deployed on the same VM as other components, like MySQL.
* The total number of instances of certain components, like Wordpress and Varnish, must be within certain limits.

**Goal:**
The goal is to minimize the total cost of the VMs, which is calculated by multiplying the price of each VM by whether it is being used or not. The problem is trying to find the cheapest way to assign the components to the VMs while meeting all the requirements and restrictions.
