**2D Bin Packing Problem**
The 2D Bin Packing Problem is a classic constraint programming problem that involves packing a set of shapes into a grid, where each shape has a specific form and size. The goal is to find a layout that packs all the shapes into the grid, satisfying certain constraints.

**Decision Variables**
The main decision variables in this problem are:
- `x[i][j]`: the index of the shape occupying the cell at row `i` and column `j` (or -1 if the cell is free)
- `y[k]`: the (index of the) base cell in the grid where we start putting the `k`th shape

**Constraints**
The key constraints in this problem are:
- Each cell in the grid can only be occupied by one shape.
- The cells that map to a shape must match the shape's form.
- A shape has exactly the right number of cells mapping to it.
- Only cells in the grid are part of the layout.

**Objective**
The objective of this problem is to find a feasible layout that packs all the shapes into the grid, satisfying the given constraints. There is no explicit objective function to minimize or maximize, but rather a satisfaction problem where the goal is to find a valid solution that meets all the constraints.
