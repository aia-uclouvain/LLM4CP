name: Layout Problem

description: The Layout Problem is a classic problem in constraint programming that involves placing a set of shapes into a grid, such that each cell in the grid is occupied by exactly one shape or is left empty. The shapes have different forms and sizes, and the goal is to find a valid layout that satisfies the given constraints.

variables: $x_{i,j}, y_k$ 
These are decision variables where $x_{i,j}$ represents the index of the shape occupying the cell at row $i$ and column $j$ (or -1 if the cell is free), and $y_k$ represents the base cell in the grid where the $k^{th}$ shape is placed.

constraints: $(y_k, x) \in table(k), \forall k \in \{1, 2, ..., nShapes\}$ 
These constraints ensure that the placement of each shape in the grid is valid, i.e., the cells occupied by the shape match the shape's form, and the shape does not overlap with other shapes or go outside the grid boundaries.

objective: The objective of the problem is to find a feasible solution, i.e., a valid layout that satisfies all the given constraints. There is no explicit objective function to minimize or maximize.
