name: Bus Driver Scheduling Problem

description: The Bus Driver Scheduling Problem is a set partitioning problem that involves scheduling bus drivers to cover a set of tasks, where each task represents a piece of work that needs to be done. The goal is to select a subset of possible shifts that covers each task once and only once, with the objective of minimizing the number of shifts used.

variables: $x_i$ where $i = 1, 2, ..., n$, $sched$
$x_i$ represents a binary decision variable that indicates whether the $i^{th}$ shift is selected (1) or not (0), and $sched$ represents a partition of the tasks into shifts. These variables determine which shifts are used to cover the tasks.

constraints: $\sum_{i=1}^{n} x_i \cdot \delta_{ji} = 1$ for all $j = 1, 2, ..., m$, $tot\_shifts = \sum_{i=1}^{n} x_i$, $tot\_shifts \geq min\_num\_shifts$, $sched \in partitions(Tasks)$, $s \in shifts$ for all $s \in parts(sched)$
These constraints ensure that each task is covered by exactly one shift, the total number of shifts used is calculated, the total number of shifts is greater than or equal to the minimum number of shifts required, and the schedule is a valid partition of the tasks into shifts.

objective: Minimize $\sum_{i=1}^{n} x_i$ or equivalently, minimize $|parts(sched)|$
The objective is to minimize the total number of shifts used to cover all tasks, which is equivalent to minimizing the number of parts in the partition of tasks into shifts.
