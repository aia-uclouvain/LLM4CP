name: Progressive Party Problem
description: The Progressive Party Problem is a classic problem in constraint programming that involves organizing a party where a set of guest boats visit a set of host boats over a series of time periods. Each host boat has a limited capacity, and each guest boat has a certain number of crew members. The goal is to find a schedule that minimizes the number of host boats needed while ensuring that each guest boat visits a host boat at each time period, and that no two guest boats meet more than once.
variables: $h, s, g, m$ - $h$ represents the set of host boats, $s$ represents the schedule of guest boats visiting host boats, $g$ represents the mapping between guest boats and host boats, and $m$ represents the mapping between guest boats and host boats at each time period. These variables determine which boats are hosts, which boats visit which hosts at each time period, and which guest boats meet at each host boat.
constraints: $|h| \geq 1, \forall p \in s . range(p) = h, \forall p \in s . \forall h \in h . p(h) = h, \forall p \in s . \forall h \in h . (\sum_{b \in preImage(p,h)} crew(b)) \leq capacity(h), \forall b1, b2 . b1 < b2 \rightarrow (\sum_{p \in s} toInt(p(b1) = p(b2))) \leq 1$ - These constraints ensure that there is at least one host boat, that the schedule only includes host boats, that each host boat is visited by itself, that the capacity of each host boat is not exceeded, and that no two guest boats meet more than once.
objective: Minimize $|h|$ - The objective is to minimize the number of host boats needed to accommodate all the guest boats while satisfying the constraints.
