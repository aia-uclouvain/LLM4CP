The Golomb Ruler problem.
The Golomb Ruler problem is a classic constraint programming problem that involves finding a set of integer marks on a ruler such that all pairwise differences between marks are distinct. The goal is to minimize the length of the ruler, which is equivalent to minimizing the maximum mark value.
The main decision variables in the Golomb Ruler problem are the mark positions, typically represented as an array of integers. Each variable represents the position of a mark on the ruler. The key constraints in the problem are: (1) all marks must be distinct, (2) all pairwise differences between marks must be distinct, and (3) the marks must be non-decreasing. Additionally, a symmetry-breaking constraint is often added to reduce the search space.
The key constraints can be described in plain language as follows: (1) no two marks can have the same position, (2) the distance between any two marks must be unique, and (3) the marks must be placed in non-decreasing order. The symmetry-breaking constraint ensures that the first difference is less than the last difference.
The objective of the Golomb Ruler problem is to minimize the maximum mark value, which represents the length of the ruler. This is equivalent to finding the shortest possible ruler that satisfies all the constraints.
