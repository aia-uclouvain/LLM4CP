N-Queens Problem
The N-Queens problem is a classic constraint programming problem that involves placing N queens on an NxN chessboard such that no two queens attack each other. A queen attacks another if they are on the same row, column, or diagonal.

Main Decision Variables
The main decision variables in the N-Queens problem are the positions of the queens on the chessboard. These are typically represented as an array of integers, where each integer represents the column in which a queen is placed in a particular row.

Key Constraints
The key constraints in the N-Queens problem are:
- Each queen must be placed in a different column (i.e., no two queens can be in the same column).
- Each queen must be placed in a different row (i.e., no two queens can be in the same row).
- No two queens can be on the same diagonal.

These constraints can be represented in plain language as follows:
- For each pair of queens, their columns must be different.
- For each pair of queens, their rows must be different (this is implicit, as each queen is placed in a different row).
- For each pair of queens, the absolute difference between their columns must not be equal to the absolute difference between their rows (this ensures that no two queens are on the same diagonal).

Objective
The objective of the N-Queens problem is to find a placement of N queens on an NxN chessboard that satisfies all the constraints. There is no explicit objective function to minimize or maximize; the goal is simply to find a feasible solution.
