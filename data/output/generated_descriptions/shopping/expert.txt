name: Shopping Puzzle
description: The Shopping Puzzle is a classic problem in constraint programming where three husbands, Arthur, Bernard, and Charles, are married to three wives, Ann, Barbara, and Cynthia. The problem states that each husband has bought a certain number of articles and each wife has bought a certain number of books, with specific price relationships between them. The goal is to determine the number of articles bought by each husband and the number of books bought by each wife.
variables: $w_{i,j}, x_i, y_j$ where $w_{i,j}$ is a binary variable indicating whether husband $i$ is married to wife $j$, $x_i$ is the number of articles bought by husband $i$, and $y_j$ is the number of books bought by wife $j$. These variables represent the marital relationships and purchasing decisions of the husbands and wives.
constraints: $\sum_{j=1}^{3} w_{i,j} = 1, \sum_{i=1}^{3} w_{i,j} = 1, x_1 = y_2 + 23, x_2 = y_1 + 11, x_i^2 = \sum_{j=1}^{3} w_{i,j} \cdot y_j^2 + 63$ for $i = 1, 2, 3$. These constraints ensure that each husband is married to exactly one wife, each wife is married to exactly one husband, and the specific relationships between the number of articles bought by each husband and the number of books bought by each wife are satisfied.
objective: satisfy, which means the objective is to find a feasible solution that satisfies all the constraints, rather than minimizing or maximizing a specific value.
