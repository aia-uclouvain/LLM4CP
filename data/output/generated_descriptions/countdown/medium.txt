Countdown Problem
The Countdown problem is a classic constraint programming problem that involves finding a way to combine a set of given numbers using basic arithmetic operators (+, -, *, /) to reach a target number.

The main decision variables in this problem are:
- `x`: an array representing the numbers in the final positions
- `perm`: an array representing the permutation of numbers for the index in `x`
- `s`: an array representing the consecutive results of the operations
- `total`: the total result of the operations
- `ops`: an array representing the operations to be performed

The key constraints in this problem are:
- Each number can be used only once
- The operations must be performed in a specific order
- The result of each operation must be used as the input for the next operation
- The final result must be equal to the target number

The objective of the problem is to find a combination of numbers and operations that results in a total that is equal to the target number. In this case, the target number is 564, and the given numbers are [3, 5, 9, 9, 10, 100]. The problem is to find a sequence of operations that combines these numbers to reach the target number.
