Alphametic Problem
The Alphametic Problem is a classic constraint programming problem where letters in a mathematical equation are replaced by digits, and the goal is to find the correct digit assignments that make the equation true.

The main decision variables in this problem are the letters (a to z) and the rad variables (rad01 to rad40), which represent the digits to be assigned to each letter. The variables a1 to z5 and a2 to z5 are used to construct the five-digit numbers represented by each letter.

The key constraints in this problem are:
- Each letter must be assigned a unique digit.
- The constructed five-digit numbers must be within the range 10000 to 99999.
- The equation defined by the rad variables must be satisfied, where each rad variable is a sum of products of digits.

The objective of this problem is to find an assignment of digits to letters that satisfies all the constraints, making the equation true. There is no explicit objective function to minimize or maximize; the goal is to find a feasible solution that meets all the conditions.
