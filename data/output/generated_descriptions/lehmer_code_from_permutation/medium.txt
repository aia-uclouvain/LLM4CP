Lehmer Code Problem
The problem is about finding the Lehmer code corresponding to a given permutation and restoring the original permutation from the Lehmer code.
The main decision variables are `x` and `y`, where `x` represents the Lehmer code and `y` represents the restored original permutation. The `x` array stores the number of subsequent values that each number in the permutation exceeds, while the `y` array stores the restored permutation.
The key constraints are that the Lehmer code `x` is calculated by counting the number of subsequent values that each number in the permutation `a` exceeds, and that the restored permutation `y` has the same Lehmer code `x`. Additionally, the restored permutation `y` must be a permutation of the numbers 1 to `n`, meaning that all numbers from 1 to `n` must appear exactly once. This is enforced by the `all_different(y)` constraint.
The problem's objective is to satisfy the constraints, meaning that the model is looking for a solution that correctly calculates the Lehmer code `x` from the given permutation `a` and restores the original permutation `y` from the Lehmer code `x`. The model does not have a specific objective function to minimize or maximize, but rather aims to find a feasible solution that satisfies all the constraints.
