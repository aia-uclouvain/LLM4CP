name: Min_n Problem
description: The Min_n problem is a constraint satisfaction problem that involves finding the minimum value of a given rank in a collection of domain variables. The rank is defined as the position of the minimum value in the sorted list of distinct values. The problem requires finding an assignment of values to the variables such that the minimum value of the specified rank is satisfied.
variables: $n, variables_{1..n}, t_{min}, rank$ - These are the decision variables where $n$ is the number of variables, $variables_{1..n}$ is an array of variables with domain 1..7, $t_{min}$ is the minimum value, and $rank$ is the rank of the minimum value. $n$ is fixed to 5, and the domain of $variables$ and $t_{min}$ is 1..7, while the domain of $rank$ is 0..n-1.
constraints: $t_{min} = variables_i, \forall i \in 1..n; first\_pos_i = 1 \leftrightarrow \forall j \in 1..i-1, variables_j \neq variables_i; rank = \sum_{i=1}^{n} (first\_pos_i = 1 \wedge variables_i < t_{min}); min\_n(t_{min}, rank, variables)$ - These constraints ensure that $t_{min}$ is one of the values in $variables$, $first\_pos$ is used to handle duplicate values, $rank$ is calculated as the number of elements larger than $t_{min}$, and the $min\_n$ constraint is satisfied.
objective: The objective of the problem is to satisfy the constraints, specifically the $min\_n$ constraint, which means finding an assignment of values to the variables such that the minimum value of the specified rank is satisfied. There is no explicit objective function to minimize or maximize.
