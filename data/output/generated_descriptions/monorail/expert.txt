name: Monorail Puzzle
description: The Monorail Puzzle is a classic problem in constraint programming where the goal is to complete a closed-circuit loop through all the stations (represented as dots) by drawing rails. The loop must pass through each station exactly once and close back on itself. The puzzle is defined on a rectangular grid, and the rails can only be drawn between adjacent stations (horizontally or vertically).
variables: $x[1..num\_steps], y[1..num\_steps], path[1..num\_steps]$ - These are the decision variables that represent the coordinates and the path of the monorail. $x[i]$ and $y[i]$ represent the row and column of the $i^{th}$ station in the path, and $path[i]$ represents the $i^{th}$ station in the path as an integer between 1 and $num\_steps$.
constraints: $\forall k \in 1..num\_given (path[a] = given[k,1] \wedge path[b] = given[k,2] \wedge (|a-b| = 1 \vee |a-b| = num\_steps-1))$, $alldifferent(path)$, $\forall s \in 1..num\_steps-1 (table([path[s], path[s+1]], valid\_connections))$, $table([path[num\_steps], path[1]], valid\_connections)$, $\forall i \in 1..num\_steps ((x[i]-1)*cols + y[i] = path[i])$ - These constraints ensure that the given hints are satisfied, all stations are visited exactly once, only valid connections are used, and the path forms a closed circuit. The last constraint links the coordinate representation with the path representation.
objective: satisfy - The objective of the Monorail Puzzle is to find a feasible solution that satisfies all the constraints, i.e., to find a closed-circuit loop that visits all the stations exactly once. There is no explicit objective function to minimize or maximize.
