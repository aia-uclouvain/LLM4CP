**Associative Magic Square Problem**

The Associative Magic Square problem is a classic constraint programming problem that involves finding a square grid of numbers, typically with unique values, that satisfy certain conditions.

**Problem Description**

Given an integer `n`, the goal is to find an `n x n` grid of numbers, where each number is between 1 and `n*n`, such that:

1. Each row, column, and diagonal sum to the same total value, which is `(n * (n*n + 1)) / 2`.
2. Each pair of numbers symmetrically opposite to the center of the grid sums to the same value, which is `n*n + 1`.
3. The grid is filled with unique numbers.

**Decision Variables**

The main decision variables in this problem are the values of the grid cells, represented by the `magic` array in the Minizinc model. Each `magic[i, j]` variable represents the value of the cell at row `i` and column `j`.

**Constraints**

The problem has several constraints:

1. **Uniqueness**: Each number in the grid must be unique, which is enforced by the `all_different` constraint.
2. **Row and Column Sums**: The sum of each row and column must be equal to the total value, which is enforced by the `forall` constraint.
3. **Diagonal Sums**: The sum of each diagonal must be equal to the total value, which is enforced by two separate constraints.
4. **Associate Value**: Each pair of numbers symmetrically opposite to the center of the grid must sum to the same value, which is enforced by the `forall` constraint.
5. **Frénicle Standard Form**: The grid must be in Frénicle standard form, which is enforced by two additional constraints.

**Objective**

The objective of the problem is to find a solution that satisfies all the constraints. There is no explicit objective function to minimize or maximize.

Note that the problem has multiple solutions for certain values of `n`, and the Minizinc model uses a search strategy to find one of these solutions. The `int_search` function is used to specify the search strategy, which in this case is a combination of `most_constrained` and `indomain_random`.
