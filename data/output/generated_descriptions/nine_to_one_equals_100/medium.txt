Nine to One Equals 100 Problem
The Nine to One Equals 100 problem is a classic constraint programming problem where the goal is to insert mathematical operators (+ or -) between the digits 9 to 1 (or 1 to 9) to make the equation equal to 100, with the objective of minimizing the number of operators used.

The main decision variables in this problem are:
- `x`: an array representing the digit sequence, where each element is a digit from 1 to 9.
- `y`: an array of numbers to be used in the equation, where each element is a number formed by concatenating consecutive identical digits in `x`.
- `ops`: an array of operations, where each element is either 1 (for addition) or -1 (for subtraction).
- `s`: an array representing the cumulative sum of operations.

The key constraints in this problem can be described as follows:
- The sequence of digits in `x` must be either increasing (for the 1 to 9 problem) or decreasing (for the 9 to 1 problem).
- Each number in `y` is formed by concatenating consecutive identical digits in `x`.
- The equation is formed by applying the operations in `ops` to the numbers in `y`, and the result must equal the target value of 100.
- The cumulative sum `s` is used to keep track of the intermediate results of the equation.

The objective of the problem is to minimize the number of operators (+ or -) used in the equation, which is equivalent to minimizing the number of non-zero elements in the `y` array. The problem is solved by finding the optimal assignment of values to the decision variables `x`, `y`, `ops`, and `s` that satisfies all the constraints and minimizes the number of operators used.
