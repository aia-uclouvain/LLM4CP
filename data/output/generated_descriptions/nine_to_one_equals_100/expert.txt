name: Nine to One Equals 100 Problem
description: The problem is to place mathematical operators (+/-) between the digits 9 to 1 (or 1 to 9) in order to get a result equal to 100, with the goal of minimizing the number of operators used. The digits can be concatenated to form numbers.
variables: $x_1, x_2, ..., x_9, y_1, y_2, ..., y_9, ops_1, ops_2, ..., ops_9, s_1, s_2, ..., s_9$. These variables represent the digit representation ($x_i$), the numbers to use ($y_i$), the operations to perform ($ops_i$), and the cumulative sum of operations ($s_i$).
constraints: $decreasing_me(x)$, $x_1 = 9$, $(not(contains(i, x)) <-> y_i = 0)$, $(contains(i,x) <-> exists(j, k) (forall(a) (x_a = i) /\ toNum\_reverse([a | a in j..k], y_i, 10.0)))$, $(s_1 = y_1 /\ ops_1 = 1) \/ (s_1 = -y_1 /\ ops_1 = -1)$, $y_i = 0 -> ops_i = 1$, $(s_i = s_{i-1} + y_i /\ ops_i = 1) \/ (s_i = s_{i-1} - y_i /\ ops_i = -1)$, $s_9 = 100$. These constraints ensure that the digits are in decreasing order, the numbers are formed correctly, and the operations are performed correctly to reach the target sum of 100.
objective: Minimize the number of operators (+/-) used, which is equivalent to minimizing the sum of the absolute values of $ops_i$.
