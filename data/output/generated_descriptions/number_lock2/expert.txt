name: Number Lock Problem
description: The Number Lock Problem is a classic puzzle where a 3-digit code must be cracked based on a set of hints. Each hint provides a 3-digit number and information about how many digits are correct and in the correct position, or correct but in the wrong position. The goal is to find the correct 3-digit code that satisfies all the hints.
variables: $x = (x_1, x_2, x_3)$, where $x$ is an array of 3 decision variables representing the 3-digit code, with each $x_i$ being a variable that can take on any digit from 0 to 9.
constraints: $\forall i \in \{1, 2, ..., m\}, check(y[i,..],x,y[i,4],y[i,5])$, where $check$ is a predicate that ensures the number of correct digits in the correct position and the number of correct digits in any position match the hints provided. Specifically, it checks that the sum of correct digits in the correct position is equal to $y[i,4]$ and the sum of correct digits in any position is equal to $y[i,5]$.
objective: The objective of the problem is to satisfy all the constraints, i.e., to find a 3-digit code $x$ that matches all the hints provided, with no specific value to minimize or maximize. The problem is a feasibility problem, where the goal is to find a solution that satisfies all the constraints.
