name: Magic Square Problem
description: The Magic Square Problem is a classic problem in constraint programming and combinatorial design. It involves creating a square grid of size n x n, where each cell contains a unique integer from 1 to n^2, such that the sum of the integers in each row, column, and diagonal is the same.
variables: $magic_{i,j}$ for $i = 1..n$ and $j = 1..n$, where $magic_{i,j}$ represents the integer value in the cell at row $i$ and column $j$ in the magic square, and is a decision variable that must be assigned a unique integer value between 1 and $n^2$.
constraints: 
$\bigwedge_{i=1}^{n} \bigwedge_{j=1}^{n} 1 \leq magic_{i,j} \leq n^2$ 
$\wedge$ 
$\bigwedge_{i=1}^{n} \sum_{j=1}^{n} magic_{i,j} = total$ 
$\wedge$ 
$\bigwedge_{j=1}^{n} \sum_{i=1}^{n} magic_{i,j} = total$ 
$\wedge$ 
$\sum_{i=1}^{n} magic_{i,i} = total$ 
$\wedge$ 
$\sum_{i=1}^{n} magic_{i,n-i+1} = total$ 
$\wedge$ 
$\bigwedge_{i=1}^{n} \bigwedge_{j=1}^{n} \bigwedge_{k=1}^{n} \bigwedge_{l=1}^{n} (i \neq k \vee j \neq l) \rightarrow magic_{i,j} \neq magic_{k,l}$, 
where the constraints ensure that each cell contains a unique integer, the sum of integers in each row, column, and diagonal is the same, and all integers from 1 to $n^2$ are used exactly once.
objective: The objective of the problem is to find a feasible solution that satisfies all the constraints, as the problem is a satisfaction problem and does not involve minimizing or maximizing a specific value.
