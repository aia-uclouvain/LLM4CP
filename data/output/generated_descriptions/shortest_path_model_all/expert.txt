name: Shortest Path Problem
description: The shortest path problem is a classic problem in graph theory and operations research that involves finding the path with the minimum total weight or cost between two nodes in a weighted graph. The problem has numerous applications in fields such as logistics, transportation, and network optimization.
variables: $x_{i,j}, rhs_i, outFlow_i, inFlow_i$ 
These variables represent the decision variables in the problem: $x_{i,j}$ is a binary variable that indicates whether there is a connection between nodes $i$ and $j$, $rhs_i$ is a variable that indicates whether node $i$ is the start or end node, and $outFlow_i$ and $inFlow_i$ are variables that represent the outflow and inflow of node $i$.
constraints: $\forall i \in 1..n: rhs_i = \begin{cases} 1 & \text{if } i = start \\ -1 & \text{if } i = end \\ 0 & \text{otherwise} \end{cases}, \forall i,j \in 1..n: x_{i,j} \geq 0, \forall i \in 1..n: outFlow_i = \sum_{j \in 1..n} x_{i,j}, \forall j \in 1..n: inFlow_j = \sum_{i \in 1..n} x_{i,j}, \forall i \in 1..n: outFlow_i - inFlow_i = rhs_i, \forall i \in 1..n: x_{i,i} = 0, \forall i,j \in 1..n: d_{i,j} = M \rightarrow x_{i,j} = 0$
These constraints represent the rules that the solution must satisfy: the first constraint sets the $rhs_i$ variable based on whether node $i$ is the start or end node, the second constraint ensures that all $x_{i,j}$ values are non-negative, the third and fourth constraints calculate the outflow and inflow of each node, the fifth constraint ensures that the outflow and inflow of each node are balanced, the sixth constraint prevents self-loops, and the seventh constraint ensures that there are no connections between nodes that are not connected in the original graph.
objective: minimize $total\_cost = \sum_{i,j \in 1..n} d_{i,j} \cdot x_{i,j}$ 
The objective of the problem is to minimize the total cost of the path, which is calculated as the sum of the weights of the edges in the path.
