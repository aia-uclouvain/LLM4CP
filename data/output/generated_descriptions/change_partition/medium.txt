Change Partition Problem
The Change Partition problem is about finding a partitioning of a sequence of variables into sets such that the number of times the partition changes between consecutive variables is minimized or equals a given value.

The main decision variables are:
- `variables`: an array of var integers representing the sequence of variables to be partitioned.
- `partitions`: an array of var sets of integers representing the partitioning of the variables.
- `nchange`: a var integer representing the number of times the partition changes between consecutive variables.

The key constraints are:
- Each variable must be assigned to exactly one partition.
- The partitioning must be such that the number of times the partition changes between consecutive variables equals the given value `nchange`.
- No partition can be empty.

The objective of the problem is to find a partitioning that satisfies the given constraints, specifically to find a partitioning where the number of changes between consecutive variables equals `nchange`. In this case, `nchange` is given as 2, so the objective is to find a partitioning with exactly 2 changes.
