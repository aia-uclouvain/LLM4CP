**Travelling Salesman Problem with Time Windows (TSPTW)**

The Travelling Salesman Problem with Time Windows is a classic constraint programming problem that involves finding the shortest possible tour that visits a set of cities and returns to the starting city, while also satisfying time window constraints for each city.

**Decision Variables**

The main decision variables in this problem are:
- `x`: an array of variables representing the circuit, where `x[i]` is the next city to visit after city `i`.
- `d`: an array of variables representing the distance between each pair of cities.
- `path`: an array of variables representing the path taken to visit each city.
- `cum_times`: an array of variables representing the cumulative time spent visiting each city.
- `wait_time`: an array of variables representing the waiting time at each city.

**Key Constraints**

The key constraints in this problem are:
- Each city must be visited exactly once.
- The tour must start and end at the same city.
- The time spent visiting each city must be within the specified time window.
- The cumulative time spent visiting each city must be increasing.
- The distance between each pair of cities must be taken into account when calculating the cumulative time.

**Objective**

The objective of this problem is to minimize the total travel time, which includes the time spent visiting each city and the waiting time at each city. Alternatively, the problem can also be formulated to minimize the total travel distance.
