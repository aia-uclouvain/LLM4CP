Building Blocks puzzle
The Building Blocks puzzle is a classic constraint programming problem where four alphabet blocks, each with a single letter on each of its six sides, are used to spell out a set of given words. The goal is to determine the arrangement of letters on each block.

The main decision variables in this problem are the `dice` array, which represents the block assignment for each letter. Each element `dice[j]` corresponds to the block number (1-4) that the `j-th` letter is assigned to.

The key constraints in this problem can be described as follows: 
- Each word must be spelled out using letters from different blocks, meaning that no two letters in a word can come from the same block.
- Each block must have exactly six letters assigned to it, ensuring that all sides of each block are utilized.
- To reduce symmetry and improve search efficiency, additional constraints are introduced to fix the block assignments for certain letters or to impose an order on the block assignments.

The objective of this problem is not to minimize or maximize a specific value, but rather to find a feasible solution that satisfies all the given constraints. In other words, the goal is to determine a valid arrangement of letters on the blocks that allows all the given words to be spelled out correctly. The `solve satisfy` statement in the MiniZinc model reflects this objective, as it searches for a solution that satisfies all the constraints without optimizing a specific objective function.
