name: Progressive Party Problem
description: The Progressive Party Problem is a classic problem in constraint programming that involves organizing a party where a set of guest boats visit a set of host boats over a series of time periods. Each host boat has a limited capacity, and each guest boat has a certain number of crew members. The goal is to find a schedule that minimizes the number of host boats needed while ensuring that each guest boat visits a host boat at each time period, and that no two guest boats meet more than once.
variables: $h, s, g$ - $h$ is a binary variable indicating whether a boat is a host, $s$ is a variable representing the host boat visited by a guest boat at a given time period, and $g$ is a variable representing the guest boats visiting a host boat at a given time period. These variables represent the assignment of host boats to guest boats at each time period.
constraints: $h_b = 1 \rightarrow s_{b,t} = b, \forall b \in B, \forall t \in T$ - a host boat must visit itself, $s_{b,t} = h \rightarrow h_h = 1, \forall b \in B, \forall h \in H, \forall t \in T$ - a guest boat can only visit a host boat, $\sum_{b \in B} g_{b,h,t} \cdot c_b \leq cap_h, \forall h \in H, \forall t \in T$ - the total crew size of guest boats visiting a host boat must not exceed its capacity, $s_{b_1,t} = s_{b_2,t} \rightarrow \sum_{t \in T} eq(s_{b_1,t}, s_{b_2,t}) \leq 1, \forall b_1, b_2 \in B, b_1 < b_2$ - two guest boats cannot meet more than once. These constraints ensure that the schedule is valid and feasible.
objective: minimize $\sum_{b \in B} h_b$ - the objective is to minimize the number of host boats needed to accommodate all guest boats.
