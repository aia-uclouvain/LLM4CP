**Problem Name:** Fractions Problem

**Description:** The Fractions Problem is a classic constraint programming problem that involves finding distinct non-zero digits to satisfy a given equation. The equation consists of three fractions, each with a numerator and a denominator, and the sum of these fractions should equal 1.

**Decision Variables:** The main decision variables in this problem are the digits A, B, C, D, E, F, G, H, and I, which represent the numerators and denominators of the three fractions. These variables are integers between 1 and 9, and each digit can only be used once.

**Constraints:** The key constraints in this problem are:

* The sum of the three fractions equals 1.
* Each digit can only be used once (all-different constraint).
* The denominators of the fractions are calculated as 10*B+C, 10*E+F, and 10*H+I.
* Symmetry-breaking constraints are used to reduce the search space, such as A*D2 >= D*D1 and D*D3 >= G*D2.
* Redundant constraints, such as 3*A >= D1 and 3*G <= D2, are used to further reduce the search space.

**Objective:** The objective of this problem is to find a solution that satisfies all the constraints, i.e., to find distinct non-zero digits that make the sum of the three fractions equal to 1. There is no explicit objective function to minimize or maximize; the goal is to find a feasible solution.
