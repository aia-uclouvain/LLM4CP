3-in-a-row puzzle
The 3-in-a-row puzzle is a classic constraint programming problem where the goal is to fill a grid with two colors, blue and white, in such a way that no three consecutive cells in a row or column have the same color, and each row and column has an equal number of blue and white cells.

The main decision variables in this problem are the elements of the 2D array `x`, where `x[i, j]` represents the color of the cell at row `i` and column `j`, with 0 indicating white and 1 indicating blue.

The key constraints in this problem are: 
- Each row and column must have an equal number of blue and white cells, which is achieved by ensuring that the sum of the colors in each row and column is equal to half the total number of cells in the row or column.
- No three consecutive cells in a row or column can have the same color, which is enforced using the `sliding_sum` constraint.

The objective of this problem is to find a feasible solution that satisfies all the constraints, rather than optimizing a specific objective function. In other words, the goal is to find a way to color the grid such that all the constraints are met, without trying to minimize or maximize any particular quantity. The solution is a 2D array representing the colored grid, where each cell is either blue (1) or white (0).
