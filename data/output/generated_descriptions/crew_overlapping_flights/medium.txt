**Crew Scheduling Problem**
The Crew Scheduling Problem is a classic constraint programming problem that involves assigning crew members to flights while satisfying various constraints.

**Problem Description**
The problem is about assigning a set of pilots to a set of flights, ensuring that each flight has the required number of pilots with the correct attributes (e.g., captain or first officer). The goal is to minimize the standard deviation of flown hours between pilots.

**Decision Variables**
The main decision variables are:
- `crew`: a 2D array where `crew[f, p]` represents whether pilot `p` is assigned to flight `f` (1) or not (0).
- `z`: the number of pilots who are assigned to at least one flight.

**Constraints**
The key constraints are:
- Each flight must have the required number of pilots.
- Each flight must have the required number of pilots with the correct attributes (e.g., captain or first officer).
- A pilot cannot be assigned to two flights that overlap in time.
- The standard deviation of flown hours between pilots should be minimized.

**Objective**
The objective is to minimize the standard deviation of flown hours between pilots, which is represented by the variable `z`. This means that the model aims to distribute the flown hours as evenly as possible among the pilots.
