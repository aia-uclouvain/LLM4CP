name: Lichtenstein Coloring Problem
description: The Lichtenstein Coloring Problem is a classic problem in graph theory and constraint programming. It involves finding the minimum number of colors required to color the communes of Liechtenstein such that no two adjacent communes have the same color. The problem is challenging due to the complex geography of Liechtenstein, with many enclaves and exclaves.
variables: $color\_communes_{1..11}, color_{1..29}, color\_used_{1..11}, n\_colors$. These variables represent the color assigned to each commune, the color assigned to each enclave/exclave, a binary variable indicating whether a color is used, and the total number of colors used, respectively.
constraints: $\forall i \in 1..11 (color\_used_i = 1 \leftrightarrow \exists c \in 1..29 (color_c = i))$, $\forall i \in 1..45 (color_{Lichtenstein_{i,1}} \neq color_{Lichtenstein_{i,2}})$, $color\_communes_1 = 1$, $\forall i, j \in 1..29 (cc_i = cc_j \rightarrow color_i = color_j)$, $\forall c \in 1..11 (\exists e \in 1..29 (color\_communes_c = color_e \wedge cc_e = c))$. These constraints ensure that a color is used if and only if it is assigned to an enclave/exclave, adjacent enclaves/exclaves have different colors, the first commune has color 1, enclaves/exclaves of the same commune have the same color, and each commune has a color that is used by one of its enclaves/exclaves.
objective: Minimize $n\_colors$, the total number of colors used to color the communes of Liechtenstein.
