Graph Coloring Problem, specifically the Lichtenstein Coloring Problem
The problem is about finding the minimum number of colors needed to color the communes of Lichtenstein such that no two adjacent communes have the same color.

The main decision variables are:
- `color`: an array of variables representing the color assigned to each enclave/exclave
- `color_communes`: an array of variables representing the color assigned to each commune
- `color_used`: an array of variables representing whether a color is used or not
- `n_colors`: a variable representing the total number of colors used

The key constraints are:
- Each enclave/exclave must be assigned a color
- Adjacent enclaves/exclaves must have different colors
- Enclaves/exclaves belonging to the same commune must have the same color
- Each commune must have a color that is used by one of its enclaves/exclaves
- The first commune (Balzers) must have color 1

The objective of the problem is to minimize the total number of colors used (`n_colors`), which is calculated by summing up the `color_used` array. The problem is solved using a constraint programming approach, where the solver searches for a solution that satisfies all the constraints while minimizing the number of colors used.
