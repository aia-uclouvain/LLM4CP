name: Not_In Constraint Demonstration
description: This problem demonstrates the usage of the not_in constraint in MiniZinc, which removes the values of a given collection from the domain of a variable. The problem is designed to showcase the functionality of this constraint.
variables: $n, values, v$ - These are the decision variables where $n$ is the size of the array, $values$ is an array of variables with domain 1..3, and $v$ is a variable with domain 1..3. $n$ determines the size of the $values$ array, $values$ is constrained to take specific values, and $v$ is a variable that must not take any value present in the $values$ array.
constraints: $cp1d(values, [1,3]), v = 2, not\_in(v, values)$ - These constraints ensure that the $values$ array is equal to $[1,3]$, $v$ is equal to 2, and $v$ does not take any value present in the $values$ array, respectively. The $cp1d$ constraint enforces array equality, the $v = 2$ constraint assigns a specific value to $v$, and the $not\_in$ constraint filters out values from $v$ based on the $values$ array.
objective: satisfy - The objective of this problem is to find a solution that satisfies all the given constraints, demonstrating the correct usage of the not_in constraint.
