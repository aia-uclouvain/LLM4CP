name: Multiple Machine Scheduling Problem

description: This problem involves scheduling a set of tasks on multiple machines, each with limited resources (memory), to minimize the end-time or the number of machines used. Each task has a duration and requires a certain amount of memory, and a machine can work on multiple tasks at the same time as long as its memory is not exceeded. A task cannot be split among machines and must be completed in one shot.

variables: $start\_time_t, end\_time_t, machine_t, machine\_used\_ram_{m,t}, machines\_used, last\_time$ 
These variables represent the start time, end time, and assigned machine for each task $t$, the amount of memory used by machine $m$ at time $t$, the number of machines used, and the last time at which any task is completed, respectively.

constraints: 
$\forall t: end\_time_t = start\_time_t + duration_t - 1$ 
$\forall m, t: machine\_used\_ram_{m,t} = \sum_{i=1}^{num\_tasks} memory_i \* (machine_i = m) \* (t \in start\_time_i .. end\_time_i)$ 
$\forall m, t: machine\_used\_ram_{m,t} \leq machines\_memory_m$ 
$\forall m: value\_precede\_chain([i | i \in 1..num\_machines], machine)$ 
These constraints ensure that the end time of each task is calculated correctly, the memory used by each machine at each time is calculated correctly and does not exceed the machine's memory, and the machines are used in a precedence order.

objective: minimize $last\_time$ or minimize $machines\_used$ 
The objective is to either minimize the last time at which any task is completed or minimize the number of machines used.
