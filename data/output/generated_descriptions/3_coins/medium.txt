3 Coins Problem
The 3 Coins problem is a classic constraint programming problem that involves flipping a series of coins to achieve a specific goal. 
The problem revolves around three coins placed in a specific order, with the objective of making them all face either heads or tails in a limited number of moves.

The main decision variables in this problem are represented by the 2D array `x`, where `x[m, i]` denotes the state of the `i-th` coin after the `m-th` move. The value of `x[m, i]` is either 0 (heads) or 1 (tails).

The key constraints in this problem can be described as follows: 
- The initial state of the coins is set to a specific configuration (tails, heads, tails). 
- Each move can only flip one coin, meaning that only one coin can change its state from one move to the next. 
- The final state of the coins must be either all heads or all tails.

The objective of the 3 Coins problem is to find a sequence of moves that satisfies the given constraints, with the goal of making all coins face either heads or tails in precisely three moves. The problem does not involve minimizing or maximizing a specific objective function, but rather finding a feasible solution that meets the specified conditions.
