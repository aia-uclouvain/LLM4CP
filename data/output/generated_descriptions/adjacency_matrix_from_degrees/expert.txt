name: Graph Realization Problem
description: The Graph Realization Problem is a classic problem in graph theory and constraint programming. Given an array of degrees, the problem is to determine whether there exists a simple graph (i.e., a graph without self-loops or multiple edges between any two vertices) where each vertex has the specified degree. In other words, the problem is to find an adjacency matrix representing a graph where the degree of each vertex matches the given array of degrees.
variables: $x_{i,j}$, where $x_{i,j}$ is a binary decision variable representing the presence (1) or absence (0) of an edge between vertices $i$ and $j$ in the graph. These variables form an adjacency matrix, where $x_{i,j} = 1$ indicates an edge between vertices $i$ and $j$, and $x_{i,j} = 0$ indicates no edge.
constraints: $\sum_{j=1}^{n} x_{i,j} = degrees_i$, $\sum_{j=1}^{n} x_{j,i} = degrees_i$, $x_{i,i} = 0$, and $x_{i,j} = x_{j,i}$. These constraints ensure that the degree of each vertex matches the given array of degrees, there are no self-loops, and the graph is undirected (i.e., the adjacency matrix is symmetric).
objective: The objective of the problem is to find a feasible solution, i.e., to determine whether there exists a graph that satisfies the given degree constraints. The problem does not involve minimizing or maximizing a specific value, but rather finding a valid realization of the graph.
