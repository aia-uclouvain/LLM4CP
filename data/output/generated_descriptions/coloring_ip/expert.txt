name: Graph Coloring Problem
description: The Graph Coloring Problem is a classic problem in computer science and operations research that involves assigning colors to the nodes of a graph such that no two adjacent nodes have the same color, with the goal of using the minimum number of colors possible. The problem is defined by an undirected loopless graph G = (V, E), where V is a set of nodes and E is a set of edges between the nodes.
variables: $x_{i,c}, u_c, obj$ where $x_{i,c}$ is a binary variable that equals 1 if node $i$ is assigned color $c$, $u_c$ is a binary variable that equals 1 if color $c$ is used, and $obj$ is an integer variable that represents the total number of colors used. These variables are used to represent the assignment of colors to nodes and the usage of colors.
constraints: $\forall i \in E: E[i,1] \neq E[i,2], \forall i \in V: \sum_{c=1}^{nc} x_{i,c} = 1, \forall i \in E, c \in 1..nc: x_{E[i,1],c} + x_{E[i,2],c} \leq u_c$ which are constraints that ensure there are no loops in the graph, each node is assigned exactly one color, and adjacent nodes are not assigned the same color.
objective: minimize $obj = \sum_{c=1}^{nc} u_c$, which represents the total number of colors used.
