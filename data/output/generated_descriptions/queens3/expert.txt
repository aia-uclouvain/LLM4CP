name: N-Queens Problem
description: The N-Queens problem is a classic problem in the field of constraint programming, where the goal is to place N queens on an NxN chessboard such that no queen attacks another. A queen can attack horizontally, vertically, or diagonally.
variables: $queens = (q_1, q_2, ..., q_n)$, where $q_i$ represents the column in which the $i^{th}$ queen is placed, and each $q_i$ is a decision variable that takes a value between 1 and n, representing the possible columns on the chessboard.
constraints: $\begin{aligned} 
&\bigwedge_{i=1}^{n} 1 \leq q_i \leq n \\
&\bigwedge_{i=1}^{n} \bigwedge_{j=i+1}^{n} q_i \neq q_j \\
&\bigwedge_{i=1}^{n} \bigwedge_{j=i+1}^{n} q_i + i \neq q_j + j \\
&\bigwedge_{i=1}^{n} \bigwedge_{j=i+1}^{n} q_i - i \neq q_j - j 
\end{aligned}$, which represent the constraints that each queen must be placed in a different column, and no two queens can be on the same diagonal.
objective: The objective of the problem is to find a feasible solution that satisfies all the constraints, as there is no explicit objective function to minimize or maximize. The goal is to find a configuration of queens on the chessboard such that no queen attacks another.
