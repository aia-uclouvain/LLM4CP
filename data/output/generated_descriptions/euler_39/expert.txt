name: Euler Problem 39
description: This problem is a classic example of a constraint programming problem, where we need to find the perimeter of a right-angled triangle with integral length sides that has the maximum number of solutions. The problem states that for a given perimeter p, we need to find the number of right-angled triangles with integral length sides a, b, and c, such that a^2 + b^2 = c^2 and a + b + c = p.
variables: $s_{i,1}, s_{i,2}, s_{i,3}, s_{i,4}$ for $i = 1..n_sols$, where $s_{i,1}, s_{i,2}, s_{i,3}$ represent the lengths of the sides of the triangle and $s_{i,4}$ represents the perimeter, which are decision variables representing the sides and perimeter of the triangle in each solution.
constraints: $s_{i,1} > 0 \land s_{i,2} > 0 \land s_{i,3} > 0 \land s_{i,1} < s_{i,4} \land s_{i,1} \leq s_{i,2} \leq s_{i,3} \land s_{i,3}^2 = s_{i,1}^2 + s_{i,2}^2 \land s_{i,1} + s_{i,2} + s_{i,3} = s_{i,4}$ for $i = 1..n_sols$, which ensure that each solution represents a valid right-angled triangle with integral length sides, and additional constraints to ensure that the solutions are distinct and that the perimeter is within the given limit.
objective: Maximize $num\_solutions = \sum_{i=1}^{n_sols} (bool2int(s_{i,1} > 0))$, which represents the total number of distinct right-angled triangles with integral length sides for a given perimeter.
