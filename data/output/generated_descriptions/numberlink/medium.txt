Numberlink problem
The Numberlink problem is a classic constraint programming problem where the goal is to connect pairs of numbered cells in a grid by drawing a path between them, such that each cell is used exactly once and the paths do not intersect.
The main decision variables in this problem are represented by the `SubM` array, which is a 2D array of size `NR x NC`, where `NR` is the number of rows and `NC` is the number of columns. Each element `SubM[I, J]` represents the pair number that the cell at position `(I, J)` belongs to.
The key constraints in this problem can be described as follows: each cell that is part of a pair must have exactly one connected neighbor that belongs to the same pair, while each interior cell that is not part of a pair must have exactly two connected neighbors that belong to the same pair. Additionally, the paths must not intersect, meaning that each cell can only be used once.
The objective of the Numberlink problem is to find a solution that satisfies all the constraints, which means finding a way to connect all pairs of cells without intersecting the paths. The problem is a satisfaction problem, meaning that the goal is to find a feasible solution that meets all the constraints, rather than optimizing a specific objective function.
