name: Lights On Puzzle
description: The Lights On Puzzle is a classic problem in the field of constraint programming. The problem consists of a grid of lights, where each light can be either on or off. The goal is to find the minimum number of moves required to turn on all the lights, given certain constraints. Each move consists of toggling a light, which affects not only the light itself but also its neighbors. The problem is defined by a grid of size n x n, where n is a given integer, and a set of rules that specify how the lights interact with each other.
variables: $x_{i,j}, d_{i,j}$, where $x_{i,j}$ is a binary variable representing whether the light at position (i, j) is toggled (1) or not (0), and $d_{i,j}$ is an integer variable representing the difference between the number of toggles and the initial state of the light at position (i, j).
constraints: $\forall i, j \in N: d_{i,j} \geq 0, \forall i, j \in N: \sum_{l \in N} x_{i,l} + \sum_{k \in N, k \neq i} x_{k,j} = 2d_{i,j} + r_{i,j}$, where the first constraint ensures that the difference between the number of toggles and the initial state of each light is non-negative, and the second constraint represents the rules of the puzzle, specifying how the lights interact with each other based on the initial state $r_{i,j}$.
objective: Minimize the total number of moves, represented by the variable $moves = \sum_{i \in N, j \in N} x_{i,j}$, which is the sum of all the toggles made to the lights.
