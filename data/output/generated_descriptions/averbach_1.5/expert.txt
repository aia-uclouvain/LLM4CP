name: Nim Game
description: The Nim game is a mathematical game of strategy where two players take turns removing one or more objects (in this case, matchsticks) from a pile. The player who removes the last object wins. The game starts with a pile of n matchsticks, and each player can remove at most max_num_to_take matchsticks at a time. The goal is to determine the winner of the game when both players play optimally.
variables: x, who, winner, z, z_cumsum. 
These variables represent the following: 
- x is an array of decision variables representing the number of matchsticks removed by each player at each turn, 
- who is an array representing which player makes each move, 
- winner is an array representing which player wins at each turn, 
- z is the total number of matchsticks removed, 
- z_cumsum is an array representing the cumulative sum of matchsticks removed at each turn.
constraints: 
z < n -> (who[i] = 1+((i-1) mod num_players) /\ x[i] in 1..max_num_to_take), 
x[i] > 0 <-> not exists(j in 1..i) ( x[j] = 0 ), 
z_cumsum[i] = sum(j in 1..i) ( x[j] ) /\ (z_cumsum[i] = n <-> winner[i] = who[i]) /\ (z_cumsum[i] != n <-> winner[i] = 0), 
z <= n. 
These constraints ensure that: 
- each player removes at least one matchstick and at most max_num_to_take matchsticks at each turn, 
- no player can remove zero matchsticks before any real move, 
- the cumulative sum of matchsticks removed is calculated correctly and the winner is determined at each turn, 
- the total number of matchsticks removed does not exceed the initial number of matchsticks.
objective: The objective of the problem is to satisfy the constraints, i.e., to find a valid sequence of moves that leads to a winner, rather than minimizing or maximizing a specific value.
