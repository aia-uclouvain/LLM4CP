**Young Tableaux Problem**
The Young Tableaux problem is a classic constraint programming problem that involves finding all possible standard Young tableaux of a given size. 

**Description**
The problem is about filling a Ferrers diagram, which is a pattern of boxes arranged in rows, with numbers from 1 to n in such a way that each number is used exactly once, and the numbers in each row and column are in non-decreasing order.

**Decision Variables**
The main decision variables in this problem are:
- `x[i, j]`: a 2D array representing the Ferrers diagram, where `x[i, j]` is the number in the `i-th` row and `j-th` column.
- `p[i]`: a 1D array representing the partition structure of the Ferrers diagram, where `p[i]` is the number of boxes in the `i-th` row.

**Constraints**
The key constraints in this problem are:
- Each number from 1 to n is used exactly once.
- The numbers in each row are in non-decreasing order.
- The numbers in each column are in non-decreasing order.
- The partition structure `p` is calculated based on the number of boxes in each row.
- The sum of the partition structure `p` equals n.
- The partition structure `p` is non-increasing, meaning that the number of boxes in each row is less than or equal to the number of boxes in the previous row.

**Objective**
The objective of this problem is to find all possible standard Young tableaux of a given size, which is achieved by satisfying all the constraints. The problem does not have a specific objective function to minimize or maximize, but rather aims to find all feasible solutions that satisfy the given constraints.
