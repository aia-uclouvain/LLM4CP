The Enigma problem.
The Enigma problem is about decoding a set of cryptic equations where each digit from 0 to 9 represents a specific letter, and the goal is to find the correct mapping between digits and letters that satisfies the given equations.
The main decision variables are represented by the 2D array `x(i, j)`, where `x(i, j) = 1` if digit `i-1` represents letter `j`. The letters are encoded as follows: 1 = O, 2 = N, 3 = E, 4 = T, 5 = W, 6 = H, 7 = R, 8 = L, 9 = V, 10 = Y.
The key constraints are: each digit is assigned to exactly one letter, each letter is assigned to exactly one digit, and the decoded equations must be satisfied. The equations are formed by replacing each digit with its corresponding letter and evaluating the resulting mathematical expressions. For example, the equation `ONE + TWO + THREE = FOUR` would be evaluated as the sum of the numerical values of the letters replacing the digits.
The problem's objective is to find a satisfying assignment of digits to letters, which means finding a mapping that satisfies all the constraints, including the equations. The model uses a satisfaction-based approach, where the goal is to find a solution that satisfies all the constraints, rather than optimizing a specific objective function.
