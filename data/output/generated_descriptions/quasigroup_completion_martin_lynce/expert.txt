name: Quasigroup Completion Problem
description: The Quasigroup Completion Problem is a classic problem in constraint programming that involves completing a partially filled Latin square, which is an n x n matrix where each row and column contains each number from 1 to n exactly once, with the additional property that each pair of elements in the matrix must be unique, i.e., for any two rows i and j, and any two columns k and l, if i ≠ j and k ≠ l, then x[i,k] ≠ x[j,l] and x[i,l] ≠ x[j,k].
variables: x - a 2D array of size n x n, where x[i,j] represents the value in the ith row and jth column of the matrix, which is a decision variable that can take any integer value from 1 to n.
constraints: ∀i ∈ {1,...,n}, ∀j ∈ {1,...,n}, x[i,j] ∈ {1,...,n} ∧ ∀i ∈ {1,...,n}, ∀k ∈ {1,...,n}, ∀l ∈ {1,...,n}, (i ≠ k ∨ j ≠ l) → x[i,j] ≠ x[k,l] - these constraints ensure that each row and column contains each number from 1 to n exactly once and that each pair of elements in the matrix is unique.
objective: The objective of the problem is to find a completion of the given partial Latin square that satisfies the quasigroup properties, i.e., there is no specific value to minimize or maximize, but rather to find a feasible solution that satisfies all the constraints.
