name: Cycle Detection Problem
description: The Cycle Detection Problem is a constraint programming problem that involves finding a permutation of a set of elements such that the permutation can be decomposed into a specified number of cycles. A cycle is a sequence of elements where each element is mapped to the next element in the sequence, and the last element is mapped back to the first element.
variables: $x = (x_1, x_2, ..., x_n)$, $num\_cycles$, where $x$ is a permutation of the set $\{1, 2, ..., n\}$, representing the mapping of each element to its image, and $num\_cycles$ is an integer variable representing the number of cycles in the permutation.
constraints: $all\_different(x)$, $cycle(num\_cycles, x)$, $num\_cycles = 2$, where $all\_different(x)$ ensures that each element in the permutation is unique, $cycle(num\_cycles, x)$ ensures that the permutation can be decomposed into $num\_cycles$ cycles, and $num\_cycles = 2$ specifies the number of cycles.
objective: The objective of the problem is to find a permutation $x$ that satisfies the constraints, specifically to find a permutation that can be decomposed into a specified number of cycles, in this case, 2 cycles. The problem does not have a specific objective function to minimize or maximize, but rather to find a feasible solution that satisfies the constraints.
