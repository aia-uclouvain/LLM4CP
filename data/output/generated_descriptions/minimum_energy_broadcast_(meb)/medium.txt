The Minimum Spanning Tree problem.
This problem is about finding the most cost-efficient way to connect a set of nodes in a network, where each connection between nodes has a certain cost associated with it.
The main decision variables are `parents` and `depths`, which represent the parent node of each node in the resulting tree and the depth of each node in the tree, respectively. The `parents` variable is a function that maps each node to its parent node, and the `depths` variable is a function that maps each node to its depth in the tree.
The key constraints are that each node can only have one parent, and the parent of a node cannot be the node itself. Additionally, a node can only be connected to another node if the cost of the link between them is not zero. The tree must also be connected, meaning that there is a path from the initial node to every other node. This is ensured by the constraint that the depth of a child node is greater than the depth of its parent node.
The objective of the problem is to minimize the total cost of the tree, which is calculated as the sum of the costs of the links between each node and its parent. The cost of each link is given by the `linkCosts` function, which takes a pair of nodes as input and returns the cost of the link between them. The problem is to find the tree with the minimum total cost that satisfies all the constraints.
