name: Minimum Spanning Tree Problem
description: The problem is a classic graph theory problem that involves finding a subset of the edges in a graph that connect all the nodes together while minimizing the total cost. The graph is represented by a set of nodes and a function that assigns a cost to each possible link between nodes. The goal is to find a tree-like structure, known as a spanning tree, that includes all the nodes and has the minimum total cost.

variables: $parents: V \rightarrow V, depths: V \rightarrow D, optVar: \mathbb{Z}$, where $V$ is the set of nodes, $D$ is the set of possible depths, and $\mathbb{Z}$ is the set of integers. These variables represent the parent node of each node in the spanning tree, the depth of each node in the tree, and the total cost of the tree, respectively. The $parents$ function assigns a parent node to each node, the $depths$ function assigns a depth to each node, and $optVar$ is the total cost of the tree.

constraints: $parents(initialNode) = initialNode, \forall (child, parent) \in parents: (child \neq initialNode) \rightarrow (parent \neq child \wedge linkCosts((parent, child)) \neq 0), \forall (child, parent) \in parents: child \neq initialNode \rightarrow depths(child) > depths(parent)$, which ensure that the initial node is its own parent, that each node has a valid parent with a non-zero link cost, and that the depth of each node is greater than the depth of its parent.

objective: minimize $optVar = \sum_{parent \in V} \max_{child \in V} (linkCosts((parent, child)) \cdot \mathbb{1}_{parent = parentI})$, where $\mathbb{1}_{parent = parentI}$ is an indicator function that is 1 if $parent = parentI$ and 0 otherwise. The objective is to minimize the total cost of the spanning tree, which is calculated as the sum of the maximum link costs between each parent node and its children.
