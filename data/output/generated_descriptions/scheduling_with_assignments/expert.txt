name: Scheduling with Assignments of Workers
description: This problem involves scheduling a set of jobs on a set of workers, where each job requires a certain number of workers and has a specific duration. The goal is to find a schedule that minimizes the earliest end time, which is the maximum end time of all jobs. The problem also considers precedences between jobs, idleness of workers, and the possibility of collecting workers for each job.
variables: start_time, end_time, assignment, earliest_end_time. These variables represent the start time and end time of each job, the assignment of workers to each job, and the earliest end time of all jobs. The start_time and end_time variables are arrays of size num_jobs, where start_time[j] and end_time[j] represent the start and end times of job j. The assignment variable is a 2D array of size num_jobs x num_workers, where assignment[j, w] is 1 if worker w is assigned to job j and 0 otherwise. The earliest_end_time variable represents the maximum end time of all jobs.
constraints: cumulative(start_time, duration, resource, num_workers), resource[j] = sum(w in 1..num_workers) (bool2int(assignment[j,w] = 1)), no_overlap(start_time[job], duration[job], start_time[j2], duration[j2]), precedences[p,1] before precedences[p,2], earliest_end_time*num_workers >= sum([duration[j]*resource[j] | j in 1..num_jobs]), if allow_idle = false then max([ end_time[j]*bool2int(assignment[j, w] = 1) | j in 1..num_jobs]) = sum(j in 1..num_jobs) ( duration[j]*assignment[j, w] ). These constraints ensure that the jobs are scheduled without overlapping, that the precedences between jobs are respected, that the earliest end time is calculated correctly, and that the idleness of workers is handled according to the allow_idle parameter.
objective: minimize earliest_end_time
