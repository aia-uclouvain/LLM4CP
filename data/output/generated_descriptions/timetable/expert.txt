name: Timetable Problem
description: The Timetable Problem is a classic problem in constraint programming that involves scheduling lectures for a set of courses in a given number of periods, subject to certain constraints such as conflicts between courses, availability of courses in certain periods, and limited room capacity.
variables: $timetable_{c,p}$, where $c$ represents a course and $p$ represents a period. These variables are binary decision variables that represent whether a lecture of course $c$ is scheduled in period $p$ or not. 
constraints: 
$\forall c_1, c_2 \in \{1, \ldots, courses\} \text{ where } c_1 < c_2: \text{ if } conflict_{c_1,c_2} = 1 \text{ then } \forall p \in \{1, \ldots, periods\}: timetable_{c_1,p} + timetable_{c_2,p} \leq 1$
$\forall c \in \{1, \ldots, courses\}, p \in \{1, \ldots, periods\}: \text{ if } available_{c,p} = 0 \text{ then } timetable_{c,p} = 0$
$\forall p \in \{1, \ldots, periods\}: \sum_{c=1}^{courses} timetable_{c,p} \leq rooms$
$\forall c \in \{1, \ldots, courses\}: \sum_{p=1}^{periods} timetable_{c,p} = requirement_c$
These constraints ensure that conflicting courses are not scheduled at the same time, courses are not scheduled in periods where they are not available, the room capacity is not exceeded, and each course has the required number of lectures.
objective: The objective of the problem is to find a feasible schedule that satisfies all the constraints, as indicated by the "solve satisfy" statement in the model. There is no explicit objective function to minimize or maximize.
