name: Coins Problem
description: The Coins Problem is a classic problem in constraint programming and operations research. It involves finding the minimum number of coins of different denominations that can be used to make all possible amounts of change from 1 to 100 units. The problem assumes that there are 6 coins with different denominations, and the goal is to determine the values and numbers of each coin to achieve this objective.
variables: $value_i, num_i, sel_{ij}, sum\_coins$ where $i \in \{1, 2, ..., 6\}$ and $j \in \{1, 2, ..., 100\}$. These variables represent the value of each coin ($value_i$), the number of coins of each denomination ($num_i$), the number of coins of denomination $i$ used to make change for amount $j$ ($sel_{ij}$), and the total number of coins used ($sum\_coins$).
constraints: $\forall i \in \{1, 2, ..., 6\}, \forall s \in \{1, 2, ..., 100\}: sel_{s,i} \leq num_i$, $\forall s \in \{1, 2, ..., 100\}: \sum_{i=1}^{6} value_i \cdot sel_{s,i} = s$, and $\forall i \in \{1, 2, ..., 5\}: value_i \cdot num_i < value_{i+1}$. These constraints ensure that the number of coins of each denomination used to make change for a particular amount does not exceed the available number of coins, that the total value of coins used to make change for a particular amount equals that amount, and that the total value of coins of each denomination is less than the total value of coins of the next denomination (a symmetry-breaking constraint).
objective: minimize $sum\_coins$
