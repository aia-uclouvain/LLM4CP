The name of the problem is the Coins problem. 
The Coins problem is about finding the minimum number of coins of different denominations that can make up any amount of change from 1 to 100 units, with each coin having a value and a limited number of coins available. 

The main decision variables are: 
- `value`: an array representing the value of each coin, 
- `num`: an array representing the number of each coin, 
- `sel`: a 2D array representing the number of each coin used to make up a specific amount of change. 

The key constraints are: 
- the number of each coin used to make up a specific amount of change cannot exceed the available number of coins, 
- the total value of the coins used to make up a specific amount of change must equal that amount, 
- and a symmetry-breaking constraint that ensures the value of each coin times the number of coins is less than the value of the next coin times the number of the next coin, to avoid redundant solutions. 

The problem's objective is to minimize the total number of coins used, which is represented by the variable `sum_coins`. The model uses a search strategy to find the solution that minimizes this objective, while satisfying all the constraints.
