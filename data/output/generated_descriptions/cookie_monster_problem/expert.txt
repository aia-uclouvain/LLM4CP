name: Cookie Monster Problem

description: The Cookie Monster Problem is a discrete optimization problem where the goal is to determine the minimum number of moves required to empty a set of cookie jars, each containing a certain number of cookies. In each move, the Cookie Monster chooses a subset of the jars and eats the same nonzero number of cookies from each jar.

variables: $x_{m,i}, z_m, num\_moves$ 
These variables represent the number of cookies left in jar $i$ after move $m$ ($x_{m,i}$), the number of cookies eaten from each jar in move $m$ ($z_m$), and the total number of moves needed to empty all the jars ($num\_moves$).

constraints: $\forall i \in \{1, \ldots, n\}: x_{1,i} = jars_i$, $z_1 = 0$, $\forall m \in \{2, \ldots, max\_moves\}: \forall i \in \{1, \ldots, n\}: (z_m \leq x_{m-1,i} \rightarrow x_{m,i} = x_{m-1,i} - z_m) \wedge (z_m > x_{m-1,i} \rightarrow x_{m,i} = x_{m-1,i})$, $\exists m \in \{2, \ldots, max\_moves\}: num\_moves = m-1 \wedge \forall i \in \{1, \ldots, n\}: x_{m,i} = 0 \wedge \forall m_2 \in \{m+1, \ldots, max\_moves\}: z_{m_2} = 0$
These constraints initialize the number of cookies in each jar, ensure that the number of cookies eaten in each move is consistent, and define the condition for the minimum number of moves required to empty all the jars.

objective: minimize $num\_moves$
