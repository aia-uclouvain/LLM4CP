The Cookie Monster Problem
The Cookie Monster Problem is about finding the minimum number of moves required to empty a set of cookie jars, where each move involves eating the same number of cookies from a subset of jars.

The main decision variables are `x` and `z`, where `x[m,i]` represents the number of cookies left in jar `i` after move `m`, and `z[m]` represents the number of cookies eaten from each jar in move `m`. The variable `num_moves` represents the total number of moves needed to empty all jars.

The key constraints are:
- Each jar starts with a certain number of cookies, and the number of cookies left in each jar after each move is determined by the number of cookies eaten in that move.
- The number of cookies eaten from each jar in each move is the same.
- The goal is to empty all jars, so the number of cookies left in each jar after the last move must be zero.
- The number of moves is minimized, so the model tries to find the smallest number of moves that satisfies the constraints.

The objective of the problem is to minimize the number of moves, `num_moves`, which represents the minimum number of times the Cookie Monster needs to eat cookies to empty all jars. The model uses a minimize statement to achieve this objective.
