**Word Design Problem**
The Word Design problem is a classic constraint programming problem that involves designing a set of words for DNA computing on surfaces. 

**Problem Description**
The problem requires finding a set of words, where each word consists of 8 symbols from the set {A, C, G, T}. The goal is to maximize the number of words in the set while satisfying certain constraints.

**Decision Variables**
The main decision variables in this problem are the words themselves, represented as functions that map each index (1-8) to a base (A, C, G, or T). In the given implementations, these variables are represented as arrays or functions, such as `x` in the PyCSP3 model or `words` in the Essence model.

**Constraints**
The key constraints in this problem can be described in plain language as follows:
- Each word must have exactly 4 symbols from the set {C, G}.
- Each pair of distinct words must differ in at least 4 positions.
- For each pair of distinct words, the reverse of the first word and the Watson-Crick complement of the second word must also differ in at least 4 positions.
The Watson-Crick complement of a word is obtained by replacing each A with T, each C with G, each G with C, and each T with A.

**Objective**
The objective of the Word Design problem is to maximize the number of words in the set while satisfying the constraints. In other words, the goal is to find the largest possible set of words that meet the specified conditions.
