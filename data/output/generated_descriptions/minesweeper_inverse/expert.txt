name: Minesweeper Inverse Problem
description: The Minesweeper Inverse Problem is a classic problem in constraint programming where the goal is to find the positions of mines in a Minesweeper game, given the number of mines in the neighborhood of each cell. The problem is the inverse of the traditional Minesweeper problem, where the positions of the mines are known and the player has to find the number of mines in the neighborhood of each cell.
variables: $r, c, X, game_{i,j}, mines_{i,j}$, where $r$ and $c$ are the number of rows and columns in the game, $X$ is a constant representing an unknown value, $game_{i,j}$ is a decision variable representing the number of mines in the neighborhood of cell $(i,j)$, and $mines_{i,j}$ is a decision variable representing whether there is a mine in cell $(i,j)$ or not.
constraints: $\forall i \in 1..r, j \in 1..c: game_{i,j} = \sum_{a=-1,0,1, b=-1,0,1}^{i+a \leq r, j+b \leq c, i+a > 0, j+b > 0} mines_{i+a,j+b}$, which ensures that the number of mines in the neighborhood of each cell is correctly calculated.
objective: satisfy, the objective is to find a solution that satisfies all the constraints, there is no explicit objective function to minimize or maximize.
